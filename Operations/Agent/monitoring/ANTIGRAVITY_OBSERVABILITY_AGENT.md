## SECTION 4 — OBSERVABILITY VIOLATION TAXONOMY (SEALED)

All violations must be classified into exactly one of these 8 categories:

### Category 1: METRIC COVERAGE VIOLATION (MCV)
A service, endpoint, screen, or domain that is missing required metric emission.

| Sub-type | Code | Description |
|---|---|---|
| Missing golden signal | MCV-GS | Service missing any of: latency (P50/P95/P99), traffic (RPS), errors (4xx/5xx), saturation (CPU/mem/queue) |
| Missing domain metric | MCV-DM | Domain-critical metric absent (e.g., `dojo.match.match_start_fail`, `billing.gateway_fail`) |
| Missing tenant metric | MCV-TM | Per-tenant request/resource/throttle metrics absent |
| Missing UI metric | MCV-UI | Flutter screen or Next.js page missing: load time, first meaningful render, user error, action success/fail |
| Missing Prometheus endpoint | MCV-PE | Service has no `/metrics` endpoint or endpoint returns no data |
| Metric naming violation | MCV-NM | Metric does not follow `domain.subsystem.component.metric_name` convention |
| High-cardinality unbounded | MCV-HC | Metric with unbounded label set (e.g., user_id, request_path as label) — must be capped |
| Missing metric contract | MCV-MC | Metric emitted but not declared in `metrics_registry.yaml` |

### Category 2: LOGGING VIOLATION (LGV)
Structured logging is absent, malformed, or non-compliant.

| Sub-type | Code | Description |
|---|---|---|
| Plain-text log | LGV-PT | Service emitting non-JSON logs in any environment |
| Missing mandatory field | LGV-MF | Structured log missing any of 12 mandatory fields (timestamp, trace_id, span_id, user_id, tenant_id, role, service, endpoint, action, status, error_code, latency_ms) |
| PII in log | LGV-PI | Personally Identifiable Information present in log body (name, email, phone, password hash) |
| Missing correlation ID | LGV-CI | Log record missing `request_id` / `trace_id` — breaks end-to-end traceability |
| Cross-tenant log leak | LGV-CT | Log record visible to wrong tenant or domain |
| Audit log mutable | LGV-AM | Audit log (auth, role change, score, payment, certification) is writable or deletable — must be append-only WORM |
| Missing audit event | LGV-AE | Required audit event not logged (see sealed audit event list in Section 5.2) |
| Log schema drift | LGV-SD | Emitted log structure diverges from declared `log_schema.json` |
| Retention violation | LGV-RV | Log tier retained shorter than required (Security/Audit: 7yr, Financial: 10yr, Ops: 2yr, Debug: 90d) |
| Missing bootstrap self-test | LGV-BS | Service does not emit test log on startup (R52.16 violation) |
| Silent UI failure | LGV-SF | UI error swallowed without log/telemetry emission |
| Anonymous complaint unmasked | LGV-AC | Anonymous complaint identity not masked at log source |

### Category 3: TRACING VIOLATION (TCV)
Distributed trace coverage is incomplete, broken, or misconfigured.

| Sub-type | Code | Description |
|---|---|---|
| Missing trace context propagation | TCV-CP | Trace context not propagated across service boundary (service-to-service, API-to-queue, API-to-DB) |
| Missing required span | TCV-SP | Required span absent for layer: UI render_time, API auth_check, Service business_exec, DB query_exec, Queue enqueue, AI inference, Payment settlement |
| Under-sampling critical flow | TCV-SC | Payment, Certification, Scoring, or Audit write sampled at < 100% |
| Over-sampling background | TCV-OB | Background flows sampled > 5% causing trace storage cost overrun |
| Orphan span | TCV-OS | Span with no parent in trace context — orphan span in production |
| Missing OpenTelemetry SDK | TCV-OT | Service not instrumented with OpenTelemetry SDK |
| Missing trace bootstrap test | TCV-BT | Service does not emit test trace on startup (R52.16 violation) |
| Trace policy drift | TCV-PD | Trace sampling policy in code diverges from declared `trace_policy.yaml` |

### Category 4: ALERT GOVERNANCE VIOLATION (AGV)
Alerts are missing, mis-configured, unowned, or not following the severity lifecycle.

| Sub-type | Code | Description |
|---|---|---|
| Missing mandatory alert | AGV-MA | Required alert not configured (see sealed mandatory alert list in Section 5.3) |
| No alert owner | AGV-AO | Alert rule has no assigned owner — every alert must have an owner |
| Missing escalation timer | AGV-ET | Alert has no auto-escalation timer configured |
| Alert severity misconfigured | AGV-SM | Alert severity does not match SEV model: SEV-0 (platform outage), SEV-1 (student/mentor), SEV-2 (partial degradation), SEV-3 (background) |
| Alert state lifecycle absent | AGV-SL | Alert does not cycle through: Open → Acknowledged → Escalated → Resolved |
| State transitions not audited | AGV-TA | Alert state change not logged to audit trail |
| Tenant alert flooding | AGV-TF | Tenant-level noisy alert flooding shared pipeline without isolation |
| Public dashboard | AGV-PD | Monitoring dashboard accessible without authentication (R51.11 violation) |
| Alert on mutable metric | AGV-AM | Alert defined on metric that is not in metrics_registry.yaml (orphan alert) |
| Missing SLO burn-rate alert | AGV-BR | SLO defined but no burn-rate alert configured for it |
| CI/CD no observability gate | AGV-CI | Post-deploy health checks and canary metric comparison absent from CI/CD pipeline |

### Category 5: SLO / ERROR BUDGET VIOLATION (SEV)
Service Level Objectives are missing, undeclared, or not enforced.

| Sub-type | Code | Description |
|---|---|---|
| Missing SLO definition | SEV-MD | Blast domain has no explicit SLO document in `slo_definitions.yaml` |
| SLO target exceeded | SEV-TE | Measured or declared performance exceeds the SLO target (latency, availability, error rate) |
| No error budget tracking | SEV-EB | SLO defined but no error budget counter / burn-rate tracking configured |
| SLO ownership unassigned | SEV-OA | SLO domain has no assigned owner |
| Error budget exhausted | SEV-EX | Error budget for a blast domain has been exhausted — feature freeze trigger not activated |
| Feature shedding order wrong | SEV-FS | Feature shedding order during SLO breach is incorrect (analytics must shed before student discussions — student discussions are never shed) |
| Missing availability SLO | SEV-AV | Required availability SLO absent: Core ≥99.9%, Dojo ≥99.95%, Auth ≥99.99% |
| SLO not in CI/CD gate | SEV-CI | SLO regression not detected or blocking in CI/CD pipeline |

### Category 6: DASHBOARD GOVERNANCE VIOLATION (DGV)
Dashboards are missing, poorly scoped, or not access-controlled.

| Sub-type | Code | Description |
|---|---|---|
| Missing mandatory dashboard | DGV-MM | One of the 5 required dashboard types absent (Executive Control, Operations Center, Assessment Integrity, User Success, Security & Abuse) |
| Blast domain not separated | DGV-BD | Ecoskiller Core and Dojo SAAS metrics/logs mixed on same dashboard panel |
| Dashboard not versioned | DGV-NV | Dashboard JSON/YAML not under version control |
| RBAC not enforced | DGV-RB | Dashboard accessible to roles beyond declared access matrix |
| Cross-tenant panel | DGV-CT | Dashboard panel shows data from multiple tenants without tenant filter |
| Aspirational dashboard | DGV-AS | Dashboard shows metrics for unimplemented features or future state components |
| Missing screen-level telemetry | DGV-SL | Dashboard has no UI screen-level performance panels (FCP, navigation time, interaction delay) |
| Missing blast-domain dashboard | DGV-BM | Any declared blast domain (Ecoskiller Core, Dojo, Trust Layer) has no dedicated Grafana dashboard |

### Category 7: INCIDENT & FORENSIC READINESS VIOLATION (IFV)
Incident infrastructure is absent, incomplete, or not forensically capable.

| Sub-type | Code | Description |
|---|---|---|
| Missing incident runbook | IFV-MR | SEV-level runbook absent for one of the 4 required SEV levels |
| Runbook missing required fields | IFV-RF | Runbook missing any of: dashboard path, metric name, normal range, alert condition, immediate action, escalation step |
| No on-call rotation | IFV-OC | No on-call rotation defined or documented |
| Missing escalation matrix | IFV-EM | No escalation chain defined for any SEV level |
| No post-incident template | IFV-PI | Post-incident review template absent |
| Forensic trail incomplete | IFV-FT | Forensic audit trail missing required fields: actor_id, policy_version, evidence_refs, hash_chain, timestamp |
| Tamper protection absent | IFV-TP | Audit logs lack WORM storage, Merkle audit tree, or quarterly integrity proof |
| Regulatory export missing | IFV-RE | Platform cannot export GDPR SAR, India DPDP, Financial Audit, or Education Audit on demand |
| Session reconstruction impossible | IFV-SR | User session timeline, mentor decision chain, certification lifecycle, or payment lifecycle cannot be reconstructed from logs |
| SIEM not integrated | IFV-SI | Wazuh SIEM not receiving security events or not integrated with alerting pipeline |
| Incident workflow not logged | IFV-WL | Any step of the 6-step incident workflow (Detect → Isolate → Notify → Mitigate → Audit → Improve) not logged |

### Category 8: BLAST DOMAIN ISOLATION VIOLATION (BDV)
Observability data is crossing blast domain boundaries — cross-contamination of telemetry.

| Sub-type | Code | Description |
|---|---|---|
| Cross-domain metric aggregation | BDV-MA | Ecoskiller Core and Dojo metrics aggregated in a single query or panel |
| Cross-tenant log access | BDV-LA | Log query returns results spanning multiple tenants |
| Cross-domain alert | BDV-CA | Alert condition references metrics from two different blast domains without explicit cross-domain scope declaration |
| Cross-domain trace | BDV-CT | Trace spans cross blast domain boundary without domain isolation tag |
| Shared log pipeline | BDV-LP | Ecoskiller Core and Dojo SAAS logs flowing through same Loki stream without domain partition |
| Arts/Commerce/Science metric cross-contamination | BDV-DC | Arts domain metrics visible in Commerce or Science domain dashboards |
| Anonymous complaint identity leak | BDV-AC | Complaint data visible in tenant-scoped query that should be identity-masked |
