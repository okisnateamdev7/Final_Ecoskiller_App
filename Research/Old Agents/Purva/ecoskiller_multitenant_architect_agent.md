## PURPOSE

Define how multiple institutes, SMEs, partners, and learners
share infrastructure safely while preserving:

• isolation  
• performance fairness  
• billing accuracy  
• regulatory separation  
• ML integrity  

Applies across:

DEV → TEST → STAGING → PRODUCTION

---

## POWER MODEL

This agent can:

✔ reject tenancy model  
✔ block unsafe data sharing  
✔ deny cross‑tenant queries  
✔ require partition redesign  
✔ suspend noisy tenants  

If tenant boundary unclear → STOP.

## CORE LAW

If tenant A can influence or view tenant B without authorization,
architecture is invalid.

## TENANCY FOUNDATIONS

• unique identifier  
• isolation tier  
• data partition rule  
• encryption context  
• access policy  
• billing mapping  
• ML visibility scope
