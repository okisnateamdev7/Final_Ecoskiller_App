## 3. MODERATION GOVERNANCE PRINCIPLES

All moderation must be:
- Policy-backed
- ML-assisted (not ML-only)
- Human-overridable
- Bias-audited
- Tenant-aware
- Appeal-supported

Each Phase = 1 Agent Cycle  
Each Phase = 10 Enforced Controls

---

## PHASE 1 — CONTENT MODERATION FOUNDATION

### 1. Content Classification Pipeline

All content classified as:
- Question
- Answer
- Feedback
- Debate
- Announcement
- Report
- Off-topic

ML Models:
- NLP Topic Classification
- Toxicity Detection
- Sentiment Analysis

---

### 2. Policy Rule Mapping

Every moderation action must reference:
- Policy ID
- Rule ID
- Severity Level (LOW / MEDIUM / HIGH / CRITICAL)

---

### 3. ML Moderation Algorithms

Mandatory models:
- Perspective-style toxicity scoring
- Hate speech detection
- Harassment detection
- Spam classification
- Contextual sentiment models

Confidence threshold < 0.7 → Human review required

---

### 4. Escalation Workflow

Levels:
- Auto-hide
- Soft warning
- Temporary lock
- Moderator review
- Trust & Safety escalation

---

### 5. Appeals System

Required:
- Appeal window
- Evidence snapshot
- Moderator rationale
- ML confidence disclosure
