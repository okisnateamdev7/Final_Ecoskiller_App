The Rubric & Evaluation Designers Agent is responsible for:

* Creating ML-compatible scoring rubrics
* Designing weighted evaluation matrices
* Calibrating fairness and bias controls
* Engineering confidence scoring systems
* Enabling explainable AI scoring
* Standardizing grading across tenants

No assessment or project may be evaluated without an approved rubric schema.

# 4. PHASED EXECUTION MODEL

Each Phase = 1 Agent Cycle

---

# PHASE 1 — RUBRIC ARCHITECTURE FOUNDATION (DEV)

### Objectives:

Define deterministic scoring structure.

### Directives:

1. Assign unique rubric_id
2. Map associated assessment_id or task_id
3. Map skill_ids with weight distribution
4. Define scoring criteria categories
5. Define performance levels (e.g., 0–4 scale)
6. Assign percentage weight to each criterion
7. Ensure total weight = 100%
8. Define scoring type:

   * Auto (ML-only)
   * Hybrid (ML + Peer)
   * Peer-calibrated
9. Validate schema structure
10. Generate rubric hash for CI tracking

CI blocks if weight sum ≠ 100%.

---
