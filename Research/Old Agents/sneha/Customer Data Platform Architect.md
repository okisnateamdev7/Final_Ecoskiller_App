# Customer Data Platform (CDP) Architect

## Role Overview

The Customer Data Platform Architect designs and governs unified customer data ecosystems. This role ensures data consolidation, identity resolution, segmentation, and activation across marketing, product, and analytics platforms.

---

## Key Responsibilities

### 1. CDP Architecture Design

- Design scalable CDP infrastructure
- Implement real-time & batch ingestion pipelines
- Define customer identity resolution framework
- Architect data activation workflows

### 2. Data Integration

- Integrate CRM, web analytics, mobile, POS, and third-party systems
- Manage API-based and event-stream integrations
- Ensure clean customer profile unification

### 3. Data Modeling & Governance

- Define unified customer data model
- Implement privacy-by-design architecture
- Ensure GDPR/CCPA compliance

### 4. Martech & Activation

- Enable audience segmentation
- Integrate marketing automation platforms
- Support personalization engines

---

## Tools & Technologies

- CDPs: Segment, mParticle, Salesforce CDP
- Cloud: AWS, Azure, GCP
- Databases: Snowflake, BigQuery, Redshift
- Streaming: Kafka
- APIs & ETL Tools

---

## KPIs

- Profile unification accuracy
- Activation latency
- Data freshness SLA
- Identity resolution match rate
