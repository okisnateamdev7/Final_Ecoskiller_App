# 3. Environment Governance Rules

The CTO Agent enforces strict separation across:

## 3.1 Development (DEV)

**Purpose**

- Rapid feature development
- ML experimentation
- Architecture prototyping

**Rules**

- Feature branches mandatory
- Mock APIs allowed
- Synthetic datasets only
- No production secrets
- Logging level: DEBUG
- CI required before merge

---

## 3.2 Testing (TEST)

**Purpose**

- Functional validation
- ML validation
- Security testing

**Rules**

- Automated integration testing
- Regression testing mandatory
- Dataset validation checks
- Performance baseline testing
- Security scans required
- Logging level: INFO

---

## 3.3 Staging (PRE-PRODUCTION)

**Purpose**

- Production simulation
- Load testing
- Final ML validation
- UAT approval

**Rules**

- Production-like anonymized data
- Infrastructure parity with production
- Performance benchmarking
- Canary release testing
- ML latency monitoring
- Logging level: WARN

---

## 3.4 Production (PROD)

**Purpose**

- Live user environment

**Rules**

- No experimental code
- Immutable infrastructure
- ML drift monitoring active
- Full observability enabled
- Incident response auto-triggered
- Logging level: ERROR
- Strict RBAC enforcement
