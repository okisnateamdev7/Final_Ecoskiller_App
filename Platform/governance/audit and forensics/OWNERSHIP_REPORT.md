## 2️⃣ PURPOSE DECLARATION

### The Problem This Agent Solves

Ecoskiller Antigravity hosts 300+ user types — students, trainers, course creators,
assessment designers, idea contributors, project executors, Dojo participants,
mentors, and institutional administrators — all producing intellectual property
simultaneously on a multi-tenant platform.

At 10M–100M scale, the following IP governance failures are structurally inevitable
without a dedicated ownership engine:

| Failure Class | Consequence |
|---|---|
| Ownership ambiguity | ROYALTY_ENGINE cannot determine payee |
| Attribution loss | Trainer's course content attributed to platform or wrong party |
| Derivative without lineage | Remixed content untraceable to original creator |
| Collaborative ownership disputes | Multiple contributors claim sole ownership |
| Cross-tenant IP theft | Trainer from Tenant A copies Tenant B's materials |
| Platform-assisted content ownership gap | AI-assisted content: who owns it? |
| Legacy preservation failure | Trainer legacy record loses attribution link |
| Certificate issuer ambiguity | Which institution issued which credential? |
| Belt and badge chain break | Achievement ownership loses trainer validator link |
| Orphaned IP | Content whose creator account is deleted or suspended |

OWNERSHIP_REPORT_AGENT eliminates every one of these through:

1. **IP Object Registration** — Every content object gets an ownership record
   at the moment of creation, before any downstream processing.

2. **Contribution Ledger** — Multi-contributor works carry weighted contribution
   records. Ownership is a proportion, not a binary.

3. **Derivative Lineage Graph** — Every remix, adaptation, or derivative work
   carries a lineage chain back to the original IP.

4. **Ownership State Machine** — Ownership has defined states:
   CLAIMED → VERIFIED → CONTESTED → RESOLVED → TRANSFERRED → ARCHIVED.

5. **Dispute Resolution Protocol** — Structured, evidence-based, GOVERNANCE_AGENT
   arbitrated, HASH_PROOF sealed dispute process.

6. **Royalty Entitlement Registry** — The authoritative source that ROYALTY_ENGINE
   queries before any payment is made.

7. **Orphaned IP Protocol** — When a creator account is deleted, suspended,
   or transferred, IP is not abandoned — it enters managed custody.

8. **Ownership Reports** — Structured, scheduled, on-demand reports for creators,
   institutions, compliance teams, and legal proceedings.

## 4️⃣ OWNERSHIP STATE MACHINE

Every IP object passes through defined ownership states:

```
OWNERSHIP_STATES:
  PENDING_REGISTRATION    Object created, ownership registration in progress
  CLAIMED                 Ownership asserted by creator, not yet verified
  VERIFIED                Ownership verified (HASH_PROOF + identity confirmed)
  JOINT_PENDING           Multi-contributor ownership, contribution shares
                          under negotiation / auto-assignment
  JOINT_VERIFIED          Multi-contributor ownership verified and locked
  CONTESTED               Dispute filed — ownership under review
  DISPUTE_RESOLVED        Dispute resolved by GOVERNANCE_AGENT
  TRANSFERRED             Ownership transferred to new holder via formal process
  LICENSED                Original ownership retained, usage rights licensed
  PLATFORM_CUSTODY        Orphaned IP under platform managed custody
  ARCHIVED                IP archived (immutable, no further commercial activity)
  REVOKED                 Ownership claim revoked (fraud determination)

STATE TRANSITIONS (ALLOWED):
  PENDING_REGISTRATION → CLAIMED           (on successful creator identity verification)
  CLAIMED → VERIFIED                       (on HASH_PROOF + COPY_PROBABILITY below threshold)
  CLAIMED → CONTESTED                      (on dispute filing by third party)
  VERIFIED → CONTESTED                     (on dispute filing after verification)
  VERIFIED → JOINT_PENDING                 (on co-contributor claim submission)
  JOINT_PENDING → JOINT_VERIFIED           (on all contributor share agreement)
  JOINT_PENDING → CONTESTED                (on contributor share dispute)
  JOINT_VERIFIED → CONTESTED               (on post-verification dispute)
  CONTESTED → DISPUTE_RESOLVED             (on GOVERNANCE_AGENT arbitration completion)
  DISPUTE_RESOLVED → VERIFIED              (if original claim upheld)
  DISPUTE_RESOLVED → TRANSFERRED           (if claim transferred to challenger)
  DISPUTE_RESOLVED → JOINT_VERIFIED        (if joint ownership determined)
  DISPUTE_RESOLVED → REVOKED               (if fraud determined)
  VERIFIED → TRANSFERRED                   (on voluntary ownership transfer)
  VERIFIED → LICENSED                      (on licensing agreement creation)
  VERIFIED → PLATFORM_CUSTODY             (on creator account deletion/suspension)
  PLATFORM_CUSTODY → TRANSFERRED           (on custody assignment to new owner)
  PLATFORM_CUSTODY → ARCHIVED             (on platform custody expiry)
  ANY → ARCHIVED                           (on platform governance archival order)
  REVOKED → ARCHIVED                       (fraud case closed)

FORBIDDEN TRANSITIONS:
  ARCHIVED → ANY                           (archived = immutable final state)
  REVOKED → VERIFIED                       (revoked ownership cannot be reinstated)
  REVOKED → CLAIMED                        (same actor cannot re-claim revoked IP)

STATE CHANGE AUTHORITY:
  PENDING_REGISTRATION → CLAIMED:          OWNERSHIP_REPORT_AGENT (automatic)
  CLAIMED → VERIFIED:                      OWNERSHIP_REPORT_AGENT (automatic, threshold-based)
  ANY → CONTESTED:                         Any authenticated platform user with evidence
  CONTESTED → DISPUTE_RESOLVED:            GOVERNANCE_AGENT only (human arbitration)
  VERIFIED → TRANSFERRED:                  Original owner + GOVERNANCE_AGENT sign-off
  ANY → PLATFORM_CUSTODY:                  OWNERSHIP_REPORT_AGENT (automatic on account event)
  ANY → ARCHIVED:                          GOVERNANCE_AGENT + COMPLIANCE_ADMIN dual approval
  ANY → REVOKED:                           GOVERNANCE_AGENT + CISO dual approval
```

## 7️⃣ AI-ASSISTED CONTENT OWNERSHIP RULES

```
AI-ASSISTED CONTENT is classified into three scopes.
Ownership determination depends on scope.

SCOPE: MINOR (ai_assistance_scope = MINOR)
  Definition: Human-authored content with AI spelling, grammar,
              or formatting assistance.
  Ownership: 100% human creator.
  Royalty: Full royalty to human creator.
  Disclosure: AI assistance note in ownership record (non-public facing).
```

## 9️⃣ DISPUTE RESOLUTION PROTOCOL

```
DISPUTE LIFECYCLE:

  STAGE 1 — FILING (0–48 hours)
    — Filer submits dispute with evidence_proof_references
    — OWNERSHIP_REPORT_AGENT:
        → Validates filing (schema + evidence completeness)
        → Creates OWNERSHIP_DISPUTE_RECORD
        → Sets ip_object_frozen = true (no further writes to IP object)
        → Notifies contested owner via NOTIFICATION_AGENT
        → Notifies GOVERNANCE_AGENT (case queued)
        → Emits: ownership.dispute.filed event

  STAGE 2 — EVIDENCE COLLECTION (48 hours – 14 days)
    — Both parties may submit additional evidence
    — Each submission = new evidence_proof_reference added to dispute record
    — OWNERSHIP_REPORT_AGENT collects and seals all evidence
    — ACCESS_LOG_TRACKER_AGENT provides access history for IP object
    — DEEP_SIMILARITY_AGENT runs comparison on contested content
    — COPY_PROBABILITY_AGENT scores copy probability for dispute evidence
    — dispute_state = EVIDENCE_COLLECTION

  STAGE 3 — REVIEW (14 days – 21 days)
    — GOVERNANCE_AGENT reviews compiled evidence
    — HUMAN_IN_THE_LOOP_REVIEWER assigned for complex cases
    — dispute_state = UNDER_REVIEW
    — Parties may not submit new evidence after Day 14

  STAGE 4 — ARBITRATION (21 days – 30 days)
    — GOVERNANCE_AGENT issues arbitration decision
    — Decision must include:
        → resolution_outcome (from defined outcomes)
        → resolution_rationale (minimum 200 chars)
        → arbitrator_id
    — decision sealed by HASH_PROOF_AGENT
    — dispute_state = ARBITRATION → RESOLVED

  STAGE 5 — RESOLUTION EXECUTION (30–35 days)
    — OWNERSHIP_REPORT_AGENT executes state transition per decision:
        UPHELD:         IP object → VERIFIED (ownership unchanged)
        OVERTURNED:     IP object → TRANSFERRED (ownership to challenger)
        JOINT:          IP object → JOINT_VERIFIED (new contribution ledger)
        REVOKED:        IP object → REVOKED, royalties suspended
        DISMISSED:      IP object → previous state restored, unfreezes
    — ip_object_frozen = false (unless REVOKED)
    — All parties notified via NOTIFICATION_AGENT

  STAGE 6 — APPEAL WINDOW (35–65 days from filing)
    — 30-day appeal window after RESOLVED
    — Appeal requires new evidence not available during original process
    — Second appeal: FORBIDDEN (one appeal maximum per dispute)
    — If no appeal by Day 65: dispute_state = CLOSED

ESCALATION RULES:
  — Disputes involving CRITICAL copy_probability_score: fast-track (7-day resolution)
  — Cross-tenant disputes: require CISO awareness notification
  — Disputes involving platform IP: require PLATFORM_SUPER_ADMIN involvement
  — Disputes with >10,000 royalty units at stake: require BILLING_SERVICE notification
```
