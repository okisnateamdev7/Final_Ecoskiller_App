# 🔒 SECTION 3 — CORE MEASUREMENT METRICS (LOCKED DEFINITIONS)

## 3.1 Self-Calibration Accuracy (SCA) — Primary Signal

```
METRIC_NAME    = Self-Calibration Accuracy (SCA)
PURPOSE:       The single most important intrapersonal skill metric —
               how accurately does a person model their own performance?
DEFINITION:    The mean absolute deviation between a participant's
               self-predicted score and their actual measured score,
               normalized and direction-annotated.
FORMULA:
  SCA_raw = (1/n) × Σ |predicted_score_i - actual_score_i| for i = 1 to n
  SCA_score = 100 - min(SCA_raw × 2.5, 100)
  (Higher SCA_score = more accurate self-prediction)

DIRECTION_ANNOTATION (required alongside SCA_score):
  PESSIMISTIC_BIAS:   participant systematically predicts below actuals
                      (predicted < actual by > 8 points rolling average)
  OPTIMISTIC_BIAS:    participant systematically predicts above actuals
                      (predicted > actual by > 8 points rolling average)
  CALIBRATED:         rolling prediction error within ±8 points of actual
  RANDOM:             high variance, no systematic direction

COLLECTION:    Before every scored scenario:
               participant submits predicted score (0–100)
               After scoring: actual score recorded
               Delta: automatically computed

MINIMUM_SAMPLE: 5 sessions before SCA is reported to participant
               10 sessions before SCA enters skill profile

SIGNIFICANCE:  SCA is both a SKILL METRIC (calibration is a learnable skill)
               and a VALIDITY SIGNAL (low SCA = lower confidence in
               participant's self-report data from SOURCE_4)

ANTI_GAMING:  Participants who consistently predict exactly their
               actual score to inflate SCA are detectable:
               variance check — predicted scores that never vary
               across sessions trigger calibration integrity flag
```

## 3.2 Behavioral Consistency Index (BCI)

```
METRIC_NAME    = Behavioral Consistency Index (BCI)
PURPOSE:       Measures whether a participant's behavioral patterns
               are stable, predictably developing, or erratically varying
               across different pressure levels and contexts
DEFINITION:    A composite index of behavioral regularity across
               low-stakes, medium-stakes, and high-stakes conditions
COMPONENTS:
  BCI_1 = Retry_rate consistency across stakes levels
  BCI_2 = Difficulty_selection consistency (independent of stakes)
  BCI_3 = Completion_rate consistency
  BCI_4 = Self_assessment_style consistency (SCA direction stability)

FORMULA:
  BCI = (1 - σ_BCI) × 100, where σ_BCI is the normalized standard
  deviation of the four component scores across conditions
  Range: 0–100 (100 = perfectly consistent across all conditions)

INTERPRETATION:
  BCI > 75:    High consistency — behavioral patterns are reliable signals
  BCI 50–75:   Moderate consistency — patterns valid with larger sample
  BCI < 50:    High context-sensitivity — behavior varies substantially
               by condition (not inherently negative — may indicate
               strong situational awareness OR instability)

NOTE:  Low BCI is not automatically penalized. It is an interpretive
       context signal. A participant with low BCI requires larger
       sample size before behavioral conclusions are drawn.
```

## 3.3 Challenge Approach Index (CAI)

```
METRIC_NAME    = Challenge Approach Index (CAI)
PURPOSE:       Measures the systematic relationship between a participant's
               voluntary challenge selections and their current capacity level
DEFINITION:    The average deviation between participant's self-selected
               difficulty tier and the AI-recommended optimal difficulty
               tier (just above current demonstrated capacity)
FORMULA:
  CAI = mean(selected_tier_i - recommended_tier_i) for i = 1 to n
  Normalized: CAI_score = ((CAI + max_deviation) / (2 × max_deviation)) × 100

INTERPRETATION:
  CAI_score > 60: Tendency toward stretch selection (positive signal)
  CAI_score 40–60: Calibrated selection (neutral — neither avoidant nor reckless)
  CAI_score < 40: Consistent under-selection relative to capacity
                  (potential FORCE_IG4 — Comfort Zone Orbit signal)

AI_RECOMMENDED_TIER:
  Computed from: current demonstrated pass rate at each tier
  Optimal tier = the tier where predicted pass rate is 55–70%
  (sufficient challenge without overwhelming — Vygotsky Zone logic)
  The recommended tier is NOT shown to participants to prevent gaming
```

## 3.4 Adaptation Velocity (AV)

```
METRIC_NAME    = Adaptation Velocity (AV)
PURPOSE:       Measures how rapidly a participant's behavior changes
               in response to specific, targeted feedback
DEFINITION:    The average number of attempts required before a
               participant's behavior measurably shifts following
               evaluator-identified behavioral gap
FORMULA:
  For each identified behavioral gap G:
    AV_G = attempts_until_measured_improvement / baseline_average_attempts
  AV = harmonic mean of all AV_G values (penalizes slow domains)
  Normalized to 0–100 (100 = very fast adaptation)

MEASUREMENT_REQUIREMENT:
  A behavioral gap must be explicitly identified by a certified mentor
  AND documented in the system before AV tracking begins for that gap.
  AV is not computed from inferred gaps — only documented ones.

ANTI_GAMING:  AV improvement without corresponding actual performance
               improvement is flagged as calibration anomaly
```

## 3.5 Temporal Behavioral Consistency Index (TBCI)

```
METRIC_NAME    = Temporal Behavioral Consistency Index (TBCI)
PURPOSE:       Measures whether a participant's behavioral commitments
               (declared in Intrapersonal Behavioral Commitment Documents)
               are followed through in their subsequent behavior
DEFINITION:    The percentage of declared behavioral commitments that
               produce measurable behavioral change within the
               commitment window (typically 14–30 days)
FORMULA:
  TBCI = (verified_commitments / total_commitments) × 100
  where a commitment is "verified" if:
    (a) The specific behavioral change was declared
    (b) Behavioral data shows measurable change in the declared direction
    (c) The change is sustained for minimum 3 subsequent sessions

SIGNIFICANCE:  TBCI is the primary validity check for SOURCE_4 data.
               High TBCI = participant's written reflections predict
               their actual behavior (high source validity)
               Low TBCI = participant's written reflections do not
               predict behavior (low source validity — SOURCE_4
               weight is automatically reduced for this participant)

MINIMUM_SAMPLE: 3 commitment cycles before TBCI is reported
```

## 3.6 Internal Gravity Resistance Score (IGRS)

```
METRIC_NAME    = Internal Gravity Resistance Score (IGRS)
PURPOSE:       The primary antigravity assessment metric —
               measures documented, evidenced resistance to
               specific identified internal gravity forces
DEFINITION:    Composite score across all attempted gravity
               resistance events, weighted by force strength
               and resistance quality
FORMULA:
  For each gravity_resistance_event GRE:
    GRE_score = (intent_clarity × 0.25)
              + (execution_match × 0.30)
              + (behavioral_evidence_strength × 0.30)
              + (sustainability_index × 0.15)

  IGRS = weighted mean of all GRE_scores
         (weighted by gravity_force_strength at time of event)

  Range: 0–100

  Where:
    intent_clarity:            How precisely was the gravity force
                               identified and the override declared? (0–100)
    execution_match:           Did behavior measurably deviate from
                               predicted gravity-driven behavior? (0–100)
    behavioral_evidence_strength: How many independent BOP sources
                               confirm the resistance? (1 source = 40,
                               2 sources = 70, 3+ sources = 100)
    sustainability_index:      Is the resistance sustained across
                               subsequent sessions? (0–100)

ANTIGRAVITY_MATCH_REQUIREMENT:
  IGRS only counts events from DECLARED GRAVITY RESISTANCE ATTEMPTS.
  An undeclared behavioral change is NOT counted as antigravity — it may
  be coincidence, natural development, or external influence.
  Declaration must precede the behavioral evidence.
```

## 5.2 Intrapersonal Behavioral Commitment Document (IBCD) — Specification

```
IBCD_NAME              = Intrapersonal Behavioral Commitment Document
IBCD_CODE              = ARTS.INTRA.IBCD
PURPOSE                = The primary intent declaration mechanism for
                         intrapersonal antigravity work.
                         Like the Intent Architecture Document in IGM and
                         the Intent Declaration in MFM, the IBCD locks
                         the participant's declared strategy BEFORE
                         behavioral evidence is collected.
                         This is the anti-gaming foundation of the entire model.

REQUIRED_SECTIONS:

  SECTION_IBCD_1: GRAVITY FORCE IDENTIFICATION
    Content:   Identify the specific internal gravity force(s) you are
               committing to resist in this cycle
    Standard:  Must use locked taxonomy names (FORCE_IG1 through FORCE_IG7)
    Evidence:  Must cite specific behavioral data from own BOP record
               (e.g., "SOURCE_2 shows my difficulty selection has not moved
               above Tier 3 in 45 days — consistent with FORCE_IG4")
    Forbidden: Vague or untethered self-descriptions
               ("I want to be braver" = REJECTED — not specific enough)

  SECTION_IBCD_2: BEHAVIORAL TARGET DECLARATION
    Content:   What specific, observable, measurable behavior will change?
    Standard:  Must be stated as observable behavioral actions:
               "I will select Tier 4 or above for a minimum of
               3 out of 5 sessions in the next 14 days"
               NOT: "I will try harder" (unobservable)
               NOT: "I will feel more confident" (internal, unmeasured)

  SECTION_IBCD_3: EVIDENCE DEFINITION
    Content:   What specific data will confirm success or failure?
    Standard:  Must reference BOP source categories and measurable thresholds
               "SOURCE_2: difficulty selection data showing ≥ 3 Tier 4+
               sessions within the 14-day window"
               "SOURCE_1: scenario completion on Tier 4 minimum 60%"

  SECTION_IBCD_4: ADAPTATION TRIGGERS
    Content:   What would cause you to revise the strategy mid-cycle?
    Standard:  Must specify conditions (not just outcomes):
               "If SOURCE_1 pass rate at Tier 4 falls below 20% by Day 7,
               I will step to Tier 3.5 (mixed) rather than abandoning
               the stretch entirely"
    Purpose:   Shows that the participant can adapt without collapsing —
               adaptation is not failure

  SECTION_IBCD_5: ANTI-COLLAPSE GUARDRAILS
    Content:   What will you commit NOT to do, even under pressure?
    Standard:  Must be specific and behavioral:
               "I will not return to exclusive Tier 2 selection, even
               after a failed Tier 4 attempt"
               "I will not skip the self-prediction step to avoid
               exposing SCA data"

  SECTION_IBCD_6: GRAVITY FORCE MECHANISM ACKNOWLEDGMENT
    Content:   Describe the mechanism by which this gravity force operates
               in your specific behavioral context
    Standard:  Shows understanding, not just naming
               Purpose: ensures participant is working from genuine insight,
               not performing the exercise

SUBMISSION_WINDOW:     Before the development cycle starts (Day 0)
IMMUTABILITY:          Locked at cycle start, cannot be edited after
EVALUATOR_ACCESS:      Assigned IBM Mentor reviews post-cycle
POST_CYCLE_RETURN:     Returned to participant with mentor annotations
                       and TBCI computation attached
```

# 🔒 SECTION 16 — THE BEHAVIORAL SOVEREIGNTY MODEL (MASTERY ARCHITECTURE)

```
PURPOSE:   This section defines what Behavioral Sovereignty (Belt 7 mastery)
           means as a complete, coherent behavioral architecture —
           the capstone concept of the Intrapersonal Antigravity Track.
```

**Behavioral Sovereignty** is the state in which a participant has developed and documented sufficient command of their own behavioral architecture that:

```
CRITERION_1:  They can accurately identify, in real time, when an internal
              gravity force has activated — naming the specific force
              from the locked taxonomy with supporting behavioral evidence.

CRITERION_2:  Their behavior measurably and consistently differs from
              what the activated gravity force would predict —
              evidenced across multiple BOP sources and time points.

CRITERION_3:  They can articulate the precise mechanism by which their
              antigravity response works — not just that it works.

CRITERION_4:  Their behavioral architecture is SUSTAINABLE —
              not dependent on high-effort conscious override at every moment,
              but increasingly integrated into their default behavioral patterns.

CRITERION_5:  They have demonstrated the capacity to FAIL GRACEFULLY
              at antigravity behaviors — to have a gravity force win
              in a specific instance, recognize it accurately afterward,
              and re-engage without identity collapse.

CRITERION_6:  Their SCA is high enough (≥82) that their self-model is
              a reliable navigational tool — they can use their own
              self-predictions as useful planning data.
```

**Behavioral Sovereignty is NOT:**
```
NOT: Perfection — gravity forces may still activate
NOT: Psychological immunity — they can still be affected by internal forces
NOT: Therapeutic completion — this is a behavioral skill, not a healing state
NOT: Permanent — behavioral architectures require maintenance
```

# 🔒 APPENDIX A — LOCKED TERMINOLOGY GLOSSARY

| Term | Definition |
|------|------------|
| Behavioral Observable Proxy (BOP) | The five-source measurement framework that infers internal behavioral tendencies from systematic patterns of external behavior |
| Intrapersonal Behavioral Architecture | The sum of an individual's systematic behavioral patterns, tendencies, and response strategies — a structural description, not a personality label |
| Self-Calibration Accuracy (SCA) | The measurable accuracy of a participant's predictions about their own performance, relative to actual measured outcomes |

| Challenge Approach Index (CAI) | Measure of the systematic relationship between voluntary challenge selection and current demonstrated capacity level |

| Intrapersonal Behavioral Commitment Document (IBCD) | The pre-cycle intent declaration document — filed and locked before behavioral evidence collection begins — specifying gravity force targets, behavioral targets, evidence definitions, adaptation triggers, and guardrails |
| Internal Gravity Force | A systematic, self-generated behavioral pattern that consistently limits performance without external coercion — described as a data pattern, never as a personality trait |
| Fixed-State Identity Pull (IG1) | Behavioral tendency to treat current capacity as a fixed identity characteristic rather than a developable current state |
| Ego-Protection Reflex (IG2) | Systematic avoidance of situations that risk visible incompetence, particularly under observation |

| Perfectionism Paralysis (IG6) | Non-initiation or non-completion driven by unattainable internal quality standards |
| Negative Self-Attribution Gravity (IG7) | Automatic attribution of failure to stable, global, internal traits rather than specific, modifiable, situational factors |
| Attribution Style | The pattern by which a person explains the causes of events — classified across three dimensions: Locus (Internal/External), Stability (Stable/Unstable), Globality (Global/Specific) |
| Behavioral Sovereignty | The mastery state (Belt 7) in which a participant has documented, sustained, multi-source evidence of command over their own behavioral architecture — not psychological perfection, but structural skill |

| Clinical Boundary | The absolute operational limit of the IBM-AGENT: behavioral skill measurement ends where psychological diagnosis, clinical assessment, or therapeutic intervention begins |
| Behavioral Signature | A participant's longitudinal multi-dimensional behavioral profile built from all BOP sources — a structural description, not a personality type |
