# SECTION B — CIA CORE PHILOSOPHY

The Class Intelligence Aggregator exists to answer one grounded question with scientific rigor, multi-layer evidence, and deep explainability:

**"What is the true intelligence profile of this class — student by student and as a collective — across all dimensions of human intelligence, and how is it growing, stagnating, or declining over time?"**

CIA answers this for every stakeholder in their language:

- **For the Teacher:** *"Which students in my class are strong in Logical Intelligence but struggling in Linguistic? Where is the class as a whole, and which teaching methods are driving measurable intelligence growth?"*

- **For the Student:** *"What are my strongest intelligence types, where am I in my class and nationally, and what specific activities will develop my weaker intelligences?"*

- **For the Parent:** *"What are my child's natural intelligence strengths? Is my child growing faster or slower than their class? What career paths align with their intelligence profile?"*

- **For the Institute:** *"What is the intelligence distribution across all our classes? Are certain teachers or programs producing stronger intelligence growth? Which student segments need intervention?"*

- **For the Recruiter/HR:** *"What is the verified intelligence profile of this candidate — not self-declared, not resume-stated — measured over time across multiple intelligence dimensions?"*

CIA rejects:
- Single-score IQ-style intelligence reduction
- Intelligence labeling that creates academic bias or determinism
- Aggregations that expose individual student data to unauthorized roles
- Black-box scores with no scientific basis or explainability
- Static intelligence measurements that don't track growth
- Intelligence claims without confidence scoring and sample validation

CIA enforces:
- 8-dimension + behavioral + real-task multi-intelligence measurement
- 4-layer measurement architecture (psychometric · behavioral · real-task · longitudinal)
- Adaptive test difficulty (Item Response Theory)
- Test-retest reliability tracking
- Population norm comparison (class · school · state · national · global)
- Confidence scoring per intelligence dimension per student
- Growth velocity tracking (improvement speed)
- Bias detection across gender, language, region, socioeconomic tier
- Teacher effectiveness correlation (intelligence growth vs. teaching input)
- Full SHAP explainability at student and class aggregate level
- Strict role-based data access (teacher sees own class only)
- Parent sees own child only — never other students

```
LAYER 1 — PSYCHOMETRIC LAYER
  Source: Ecoskiller adaptive intelligence tests
  Method: Computer-Adaptive Testing (CAT) via Item Response Theory (IRT)
  Accuracy: Medium (good starting point)
  Frequency: Initial assessment + quarterly re-test
  Reliability: Test-retest reliability coefficient tracked per dimension

  Source: Platform behavioral signals (activity logs · dojo matches ·
          project execution · peer collaboration · challenge completion)
  Method: ML inference from behavioral sequence (LSTM)
  Accuracy: High (behavioral data reveals true intelligence patterns)
  Frequency: Continuous (event-driven)
  Reliability: Cross-validated against Layer 1 scores

LAYER 3 — REAL TASK EVIDENCE LAYER
  Source: Project deliverables · championship results · dojo scenarios ·
          integration-verified work artifacts · mentor evaluations
  Method: AI scoring + mentor calibration + outcome analysis
  Accuracy: Very High (performance on real tasks is strongest signal)
  Frequency: Per task / project / match completion
  Reliability: Highest validity — grounded in actual performance

LAYER 4 — LONGITUDINAL GROWTH LAYER
  Source: Historical scores across all 3 layers over time
  Method: Mixed-Effects Regression (individual + class-level random effects)
  Accuracy: Highest (growth trajectory is most predictive signal)
  Frequency: Computed monthly per student, annually per cohort
  Reliability: Requires minimum 6 months of data for stable estimate
```

# SECTION E — CONTRACT GATE SYSTEM

```
Student Eligibility Gate:
  student_verified = TRUE
  enrolled_in_class = TRUE
  minimum_test_attempts ≥ 1 per dimension
  minimum_activity_window ≥ 14 days
  anomaly_flag = FALSE

Class Aggregation Gate:
  minimum_students_in_class ≥ 5
  (below 5 → no class aggregate published — individual privacy protection)
  active_student_participation ≥ 60% of class enrolled

Score Confidence Gate:
  confidence < 0.40 → "Insufficient data" — no score published
  confidence 0.40–0.65 → LOW band — shown with caution flag
  confidence 0.65–0.80 → MEDIUM band — standard display
  confidence ≥ 0.80 → HIGH band — full confidence display
  confidence ≥ 0.90 + 3+ layer sources → VERIFIED band

Test-Retest Reliability Gate:
  Per dimension: test-retest correlation coefficient computed
  IF ICC (Intraclass Correlation Coefficient) < 0.70 → score flagged UNRELIABLE
  UNRELIABLE scores displayed with warning — cannot trigger academic decisions

Bias Audit Gate:
  Quarterly bias audit must pass before model redeployment
  Fairness delta > threshold on any protected attribute → HOLD → governance review

Anti-Gaming Gate:
  Repeated identical test-taking pattern → anomaly_flag + hold
  Implausibly rapid score spike (>200 points in 7 days) → FLAG + review
  Score inconsistency across layers > 2 standard deviations → confidence penalty
```

# SECTION G — 12-LANE INTELLIGENCE SCORING ARCHITECTURE

## LANE C1 — Logical-Mathematical Intelligence Lane (LMIL)

**What it measures:** Reasoning, pattern recognition, systematic problem-solving, mathematical thinking, deductive and inductive logic.

**Data Sources:**
- Adaptive IRT tests: logic puzzles, pattern sequences, mathematical reasoning
- Coding task performance (Integration Hub: GitHub complexity analysis)
- Dojo scenarios involving structured argumentation and evidence analysis
- Championship performance in logic/coding-based competitions

**Scoring Algorithm:**

```
LMIL_score =
  (IRT_theta_L1_normalized × 0.35) +
  (behavioral_reasoning_signal_L2 × 0.25) +
  (real_task_performance_L3 × 0.30) +
  (longitudinal_growth_L4 × 0.10)

IRT Normalization:
  IRT theta (typical range −3 to +3) → linear transform → EIS (0–1000)
  Population mean theta → EIS 500
  1 SD above mean → EIS ~650
  2 SD above mean → EIS ~800

Adaptive Difficulty Enforcement:
  Item difficulty adjusts after every response (CAT algorithm)
  Final score = Maximum Likelihood Estimate (MLE) of true ability
  Standard Error tracked → feeds confidence score

Behavioral Signal for Logical Intelligence:
  Platform activity pattern complexity
  Decision quality in Dojo pressure scenarios
  Project task sequencing and planning quality

Real Task Signal:
  Code commit complexity (GitHub integration)
  Mathematical reasoning in project deliverables
  Championship logic problem pass rate
```

**Output:** LMIL_EIS (0–1000) · Confidence (0–1) · Percentile (class/school/state/national)

## LANE C2 — Linguistic Intelligence Lane (LIL)

**What it measures:** Written and oral communication quality, vocabulary depth, narrative structure, persuasive ability, storytelling.

**Data Sources:**
- Adaptive IRT tests: essay analysis, vocabulary, reading comprehension, grammar
- Dojo Group Discussion match performance (Group Discussion Engine)
- Written project reports and deliverables (AI NLP scoring)
- Social feed post quality analysis (optional — consent-gated)
- Championship debate and writing performance

**Scoring Algorithm:**

```
LIL_score =
  (IRT_theta_L1_normalized × 0.30) +
  (dojo_gd_score_L3 × 0.35) +           // GD is the highest-validity linguistic measure
  (written_work_nlp_score_L3 × 0.25) +
  (longitudinal_growth_L4 × 0.10)

NLP Scoring Pipeline:
  AI analyzes: vocabulary richness · sentence complexity · coherence ·
               argument structure · grammatical accuracy · persuasion markers
  Scores normalized against grade-level population norms
  Language bias check: non-native language penalty removed (normalized)

Dojo GD Signal:
  Peer + mentor scoring from Group Discussion engine
  Adjusted for scorer calibration (Dojo T3 LOCK)
  Anomaly-checked (T9 collusion detection)
```

**Output:** LIL_EIS (0–1000) · Confidence (0–1) · Percentile

**Data Sources:**

```
IPIL_score =

```

**Output:** IPIL_EIS (0–1000) · Confidence (0–1) · Percentile

## LANE C5 — Intrapersonal Intelligence Lane (INTIL)

**What it measures:** Self-awareness quality, goal-setting consistency, self-reflection depth, decision quality under uncertainty, emotional self-regulation.

**Data Sources:**
- Self-assessment consistency scores (comparing self-reported vs. actual performance)
- Goal-setting and completion pattern (platform habit system)
- Intrapersonal IRT tests: decision simulation, reflection quality prompts
- Behavioral consistency index (BHCS from PPM feeds this lane)
- Dojo pressure scenario self-management behavior

**Scoring Algorithm:**

```
INTIL_score =
  (self_assessment_accuracy_index_L2 × 0.30) +
  (goal_completion_consistency_L2 × 0.25) +
  (irt_decision_simulation_L1 × 0.25) +
  (pressure_scenario_self_mgmt_L3 × 0.20)

Self-Assessment Accuracy Index:
  Computed as: 1 − |self_rating − objective_rating| / objective_rating
  High accuracy (self-awareness) → high score
  Systematic over-confidence or under-confidence → lower score

Goal Completion Signal:
  Pattern: goals set vs. goals completed vs. revision quality
  Adaptive goal-setting (realistic re-calibration) = positive signal
```

**Output:** INTIL_EIS (0–1000) · Confidence (0–1) · Percentile

# SECTION H — CLASS INTELLIGENCE MAP FORMULA (IMMUTABLE)

## H1 — Student Intelligence Profile (SIP) Composite

```
For each student s, for each dimension d ∈ {C1..C12}:

EIS(s, d) = Σ [Layer_Score(l, d) × Layer_Weight(l, d) × Confidence(l, d) × Decay(l, d)]
              for l ∈ {L1, L2, L3, L4}

Layer Weights (LOCKED — Cannot change without version bump):

  L1 Psychometric (IRT test)     — Base weight: 25–55% (varies by dim)
  L2 Behavioral inference        — Base weight: 15–30%
  L3 Real task evidence          — Base weight: 20–40%
  L4 Longitudinal growth bonus   — Base weight: 10–20%

  Exact per-dimension weights defined in SECTION G per lane.
  Deviation from lane-specified weights: FORBIDDEN.

Temporal Decay:
  L1 test result > 90 days  → decay 0.90
  L1 test result > 180 days → decay 0.75
  L1 test result > 365 days → decay 0.55 (strong push to retest)
  L2 behavioral signal > 30 days without new activity → decay 0.85
  L3 real task > 180 days without new task → decay 0.80

Confidence Computation:
  confidence(s, d) = weighted average of layer-level confidence values
  Layer confidence = f(sample_size, IRT_SE, inter_rater_reliability, ICC)
```

## H2 — Class Intelligence Map (CIM) Aggregation

```
For each class C, for each dimension d:

CIM_avg(C, d) = Σ EIS(s, d) for all s in C / |C_active|

CIM_std(C, d) = std_dev(EIS(s, d) for all s in C)

CIM_distribution(C, d) = histogram of EIS(s, d) in buckets:
  [0–200] [201–400] [401–500] [501–600] [601–700] [701–800] [801–900] [901–1000]

Class Growth (period P):
  CIM_growth(C, d, P) = CIM_avg_now(C, d) − CIM_avg_period_ago(C, d, P)

At-Risk Flag:
  Student s in class C is AT-RISK for dimension d if:
    EIS(s, d) < (CIM_avg(C, d) − 1.5 × CIM_std(C, d))
    AND EIS(s, d) < national_25th_percentile(d)

Gifted Flag:
  Student s is GIFTED for dimension d if:
    EIS(s, d) > national_90th_percentile(d)
    AND confidence(s, d) ≥ 0.70
```

## H3 — Teacher Effectiveness Correlation

```
TEC(teacher_t, class_C, period_P) =
  Pearson_R(teaching_input_vector, intelligence_growth_vector)

teaching_input_vector:
  = [activities_run, scenario_types, teaching_hours, methods_used,
     dojo_sessions_facilitated, project_mentoring_count, ...]

intelligence_growth_vector:
  = [CIM_growth(C, d, P) for d ∈ all active dims]

Interpretation:
  TEC ≥ 0.60 → Strong positive teaching effectiveness
  TEC 0.30–0.59 → Moderate correlation
  TEC < 0.30 → Weak or inconclusive correlation
  TEC < 0 → Negative correlation (investigate curriculum)

Privacy Rule:
  TEC displayed only to: Institute Admin + Platform Admin
  NEVER shown to teachers about their own class (conflict of interest)
  Teachers see student growth reports — not their own TEC score
```

## H4 — Career Intelligence Mapping Formula

```
CAREER_MATCH(student_s, career_c) =
  dot_product(EIS_vector(s), career_intelligence_weight_vector(c))
  normalized to [0, 1]

career_intelligence_weight_vector:
  Pre-defined per career cluster, validated by:
  - Occupational intelligence research (O*NET, NSDC, etc.)
  - Historical Ecoskiller placement outcome data

  Updated: Annually via governance board review

Top career matches:
  Ranked by CAREER_MATCH score

  Displayed with: driving intelligence dimensions (SHAP)
```

# SECTION I — ML MODEL SPECIFICATION

## I1 — Model Architecture Stack

```
PSYCHOMETRIC ENGINE:    Computer-Adaptive Testing (CAT) with Item Response Theory
                        → 2-Parameter Logistic (2PL) IRT model
                        → Maximum Likelihood Estimation for ability (theta)
                        → Standard Error tracking per item administration
                        → Item bank calibration via EM algorithm

BEHAVIORAL INFERENCE:   LSTM Sequence Model
                        → Encodes temporal behavioral event sequences
                        → Predicts behavioral intelligence signals
                        → Trained on platform activity logs + ground truth labels

NLP SCORING ENGINE:     Transformer-based (BERT-fine-tuned) text quality scorer
                        → Scores written outputs for linguistic, creative dimensions
                        → Age/level-normalized against grade population

LONGITUDINAL MODEL:     Mixed-Effects Regression (lme4-equivalent)
                        → Fixed effects: time, curriculum, teaching inputs
                        → Random effects: student-level, class-level
                        → Growth curves per student per dimension

CAREER MAPPING:         Embedding similarity model
                        → Career intelligence vectors from validated occupational data
                        → Student EIS vector similarity scoring

EXPLAINABILITY:         SHAP (mandatory on all student-facing and teacher-facing surfaces)
                        → Feature-level attribution per dimension
                        → Population context anchoring

BIAS DETECTION:         Fairlearn + custom linguistic bias detection
                        → Quarterly audit mandatory
```

## I2 — Item Response Theory (IRT) Implementation

```
Model: 2-Parameter Logistic (2PL)
  P(correct | theta, a, b) = 1 / (1 + exp(−a × (theta − b)))
  where:
    theta = student's true ability (latent trait)
    a     = item discrimination parameter (how well it separates ability levels)
    b     = item difficulty parameter (ability level where P(correct) = 0.5)

Item Bank Requirements per dimension:
  Minimum 200 calibrated items per intelligence dimension
  Item calibration: pilot tested on minimum 500 students
  Ongoing item quality monitoring: item fit statistics (chi-square, RMSEA)
  Retired items: automatically flagged when fit statistics deteriorate

Adaptive Algorithm:
  Start: Medium difficulty item (b ≈ 0)
  After each response: update theta estimate (Bayesian update or MLE)
  Next item: b closest to current theta estimate
  Termination: when SE(theta) < 0.30 OR minimum 15 items answered

Test-Retest Reliability:
  ICC (Intraclass Correlation Coefficient) computed per dimension
  ICC ≥ 0.80 = Excellent reliability
  ICC 0.70–0.79 = Acceptable
  ICC < 0.70 = Flagged → displayed with UNRELIABLE warning
  ICC recomputed: after each major item bank update

Population Norms:
  Percentile norms maintained at: class · school · state · national levels
  Norm sample requirements: minimum 1,000 students per norm group
  Norm refresh: quarterly (as user base grows)
```

## I3 — Model Training Governance

```
Training Data Requirements:
  IRT item bank: minimum 500 pilot students per item
  Behavioral LSTM: 6+ months of platform activity logs
  Longitudinal model: minimum 2 academic years of cohort data
  Career mapping: validated occupational intelligence weights + outcome data

Retraining Schedule:
  IRT item bank: Updated quarterly with new pilot data
  Behavioral LSTM: Monthly incremental update
  NLP scorer: Quarterly fine-tuning on new student outputs
  Longitudinal model: Annual full retrain + semester-level update
  Career mapping: Annual update with governance board review

Performance Gates (must pass before deploy):
  IRT: Item fit statistics pass (chi-square p > 0.05, RMSEA < 0.06)
  Behavioral LSTM: Validation MAE < 5% of EIS scale
  NLP scorer: Inter-rater agreement with human experts κ ≥ 0.75
  Longitudinal: Model ICC improvement vs. baseline ≥ 0.05
  All dimensions: ICC ≥ 0.70 for all 12 dimensions
  Bias audit: Fairness delta < 0.05 on all protected attributes
  SHAP: 100% completeness on all student-facing predictions
  Career mapping: Correlation with actual career outcomes ≥ 0.60

Failure on any gate → STOP DEPLOYMENT → governance board review
```

## I4 — Bias Detection & Fairness Framework

```
Protected Attributes Monitored:
  gender
  primary_language (linguistic bias detection — critical for LIL)
  region / state
  institute_type (private / public / rural / urban)
  socioeconomic_tier (if available — optional)
  age_cohort

Specific Bias Risks per Dimension:
  LIL (Linguistic): Language-of-test bias → non-native speakers penalized
    Mitigation: Score normalized relative to language-matched population
  KIL (Kinesthetic): Gender bias in sports-adjacent tasks
    Mitigation: Sport-neutral kinesthetic task alternatives required
  BIIL (Behavioral): Socioeconomic bias (reliability may reflect access, not intelligence)
    Mitigation: Context-aware scoring adjustments for verified low-resource contexts
  ERIL (Emotional): Cultural expression bias (emotional display norms vary by culture)
    Mitigation: Cultural norm calibration per region

Quarterly Bias Audit:
  Demographic Parity Difference (DPD) < 0.05 per dimension
  Equal Opportunity Difference (EOD) < 0.05 per dimension
  Calibration by group: equivalent prediction accuracy per protected group
  Linguistic bias report: LIL scores by primary language
  Violation → remediation required → governance board approval before redeploy
```
