# SECTION B — SDM CORE PHILOSOPHY

The Student Distribution Model answers the most operationally critical question for every institute and for the platform:

**"Across the entire student population — who is where, how are they distributed across every measurable dimension, which segments need what intervention, and how is the distribution shifting over time?"**

It answers this at every layer of granularity:

- **For the Institute Admin:** *"Our top quartile is strong in Digital Intelligence but our bottom 40% is at critical risk for dropout within 6 months. Our Class 10 cohort has shifted toward higher Placement Readiness vs. last year's batch. Intervention required in 3 departments."*

- **For the Academic Board:** *"Here is the full talent distribution of this academic year's intake. Compared to last year's batch, intelligence breadth has increased but Linguistic proficiency has declined. Here are the students most at risk of academic disengagement."*

- **For the Department Head:** *"The Science stream students are performing above national median in Logical-Mathematical Intelligence but below median in Interpersonal skills. 18 students are in the at-risk dropout zone."*

- **For the Recruiter:** *"This institute's top quartile by Placement Readiness Score has a strong concentration in Data/Analytics roles. Here is the talent density map of verified candidates across skill clusters for your hiring cycle."*

- **For the Platform Analytics Engine:** *"National student distribution by intelligence dimension, skill belt level, placement probability, and engagement cluster — updated daily."*

SDM rejects:
- Student ranking that labels individuals publicly in degrading ways
- Distribution analysis that exposes individual student data without authorization
- Segmentation that becomes self-fulfilling prophecy through academic gatekeeping
- Static cohort snapshots that ignore longitudinal drift
- Population analysis without confidence bounds and sample size validation

SDM enforces:
- Multi-dimensional simultaneous distribution across all signal axes
- Confidence-bounded percentile computation with sample size gates
- Longitudinal cohort tracking (batch-over-batch comparison)
- Dropout / disengagement survival analysis with early warning alerts
- Gifted identification with multi-source corroboration requirement
- At-risk identification with intervention priority ranking
- Talent cluster mapping (multi-dimensional segment discovery via GMM)
- Strict role-based access (institute sees own students only)
- Individual student data never exposed in population-level views
- Bias audit on all segmentation boundaries
- Full audit trail on every distribution computation

# SECTION F — 10-AXIS DISTRIBUTION ENGINE ARCHITECTURE

## AXIS D1 — Intelligence Distribution Axis (IDA)

**What it distributes:** The full 12-dimension EIS vector from CIA, aggregated into population distributions per dimension and as a composite.

**Source:** CIA `StudentIntelligenceProfile`

**Distribution Computation:**

```
Per dimension d, per population scope S:

  IDA_distribution(S, d) = {
    mean:       avg(EIS(s, d)) for s in S
    median:     median(EIS(s, d))
    std_dev:    std(EIS(s, d))
    skewness:   skewness(EIS(s, d))      // detect bimodal populations
    kurtosis:   kurtosis(EIS(s, d))
    percentiles: {p10, p25, p50, p75, p90, p95, p99}
    histogram:  buckets of width 50 from 0–1000
    sample_size: count(s in S with confidence(s,d) ≥ 0.40)
    CI_95:      bootstrap 95% confidence interval on median
  }

Composite Intelligence Score (IDA_composite per student):
  IDA_composite(s) = weighted_avg(EIS(s, d) × demand_weight(d))
  demand_weight(d) = market relevance of each intelligence type for student's domain

Skewness Alert:
  IF skewness > 1.5 or < −1.5 → bimodal risk flag
  → Visual alert in institute dashboard: "Class may have two distinct learning groups"

Benchmark Overlay:
  IDA_distribution plotted against:
  → State norm (from NationalNormBank)
  → National norm
  → Peer institute (anonymized, opt-in)
```

## AXIS D3 — Placement Readiness Distribution Axis (PRDA)

**What it distributes:** Distribution of student placement probability scores (SPPS_90d from PPM) across the population.

**Source:** PPM `PlacementProbabilityScore`

**Distribution Computation:**

```
PRDA_score(s) = PPM.SPPS_90d(s)  [0–1 probability scale → ×1000 for display]

PRDA_distribution(S) = {
  mean, median, std, percentiles, histogram
  of PRDA_score across population S
}

Readiness Bands (LOCKED — thresholds require version bump to change):
  BAND_READY:         PRDA ≥ 0.70  → "Placement Ready"
  BAND_DEVELOPING:    PRDA 0.40–0.69 → "Developing"
  BAND_AT_RISK:       PRDA 0.20–0.39 → "Placement At Risk"
  BAND_CRITICAL:      PRDA < 0.20  → "Critical Intervention Required"

Band Rate Computation:
  ready_rate(S)      = count(PRDA ≥ 0.70) / |S|
  developing_rate(S) = count(0.40 ≤ PRDA < 0.70) / |S|
  at_risk_rate(S)    = count(0.20 ≤ PRDA < 0.40) / |S|
  critical_rate(S)   = count(PRDA < 0.20) / |S|

Institute Alert:
  IF critical_rate > 0.20 (>20% students in critical band) → ALERT sent to Institute Admin
  IF at_risk_rate + critical_rate > 0.40 → escalation to Academic Board notification
```

## AXIS D7 — Dropout Risk Distribution Axis (DRDA)

**What it distributes:** Distribution of student dropout probability — the most operationally critical distribution for institute intervention planning.

**Source:** Survival Analysis model (Cox Proportional Hazards + Random Survival Forest)

**Survival Analysis Model:**

```
Hazard Function:
  h(t | X) = h0(t) × exp(β · X)
  where X = feature vector per student, t = time since enrollment

Feature Vector X for Dropout Model:
  x1  days_since_last_activity
  x2  streak_broken_count_last_30d
  x3  beda_score_trend (7d vs. 30d)
  x4  course_completion_rate
  x5  prda_score (placement readiness)
  x6  social_network_degree (connected vs. isolated)
  x7  challenge_participation_rate
  x8  dojo_match_frequency_30d
  x9  peer_project_count
  x10 mentor_interaction_count
  x11 academic_assessment_missed_count
  x12 day_of_semester (early semester → lower risk baseline)
  x13 ida_composite (lower intelligence score = higher risk)
  x14 family_income_band (if available — optional)

Dropout Risk Levels (LOCKED thresholds):
  RISK_CRITICAL:   dropout_probability_30d ≥ 0.70  → Immediate alert
  RISK_HIGH:       dropout_probability_30d 0.50–0.69 → Alert within 48h
  RISK_MODERATE:   dropout_probability_30d 0.30–0.49 → Monitor weekly
  RISK_LOW:        dropout_probability_30d 0.10–0.29 → Periodic check
  RISK_MINIMAL:    dropout_probability_30d < 0.10  → No action needed

Survival Curve Output:
  S(t) = P(student remains active past time t)
  Output: S(30), S(60), S(90), S(180) per student

Early Warning Cascade:
  RISK_CRITICAL detected:
    → Immediate push notification to Institute Admin (within 15 min)
    → Teacher alert for relevant class
    → Intervention triage queue entry created
    → Student receives re-engagement push notification (platform-level)

DRDA Distribution:
  Histogram of dropout_probability_30d across population
  Critical zone rate: what % of institute is in RISK_CRITICAL?
  IF critical_zone_rate > 0.10 → Institute-level alert + escalation
```

## AXIS D9 — Multi-Dimensional Talent Cluster Axis (MDTCA)

**What it produces:** Discovered student talent segments using GMM clustering across all active axes simultaneously.

**ML Architecture:**

```
INPUT FEATURE SPACE:
  [D1_composite, D2_composite, D3_score, D4_score,
   D5_score, D6_score, D7_inverted, D8_score]
  Standardized: Z-score normalization per feature

MODEL: Gaussian Mixture Model (GMM)
  Components: K = 6–10 (optimal K selected via BIC)
  Covariance: full covariance matrix per component
  Output: soft cluster membership probabilities per student

CLUSTER VALIDATION:
  Silhouette score ≥ 0.40 required for publication
  Davies-Bouldin index computed (lower = better separation)
  Cluster stability test: re-run with 80% sample → check cluster agreement

CLUSTER LABELING PROCESS:
  Cluster centroids inspected by governance-approved analytics team
  Human-assigned plain-language names (not algorithmic)
  Labels must not be stigmatizing or deterministic

PREDEFINED CLUSTER ARCHETYPE EXAMPLES (v1.0.0 seed labels):
  These are starting-point archetypes — actual clusters emerge from data:

  Archetype 1: "Intelligence Leaders"
    High D1 (Intelligence) + Moderate D3 (Placement) + High D8 (Social)
    
  Archetype 2: "Skill Specialists"
    High D2 (Skill) + High D3 (Placement) + Low D1 (Intelligence composite)
    
  Archetype 3: "Championship Champions"
    High D6 (Competition) + Moderate D1 + Moderate D3
    
  Archetype 4: "Rising Performers"
    Moderate all axes + High growth velocity on D1 and D2
    
  Archetype 5: "Consistent Engagers"
    High D5 (Behavioral) + Moderate D2 + Moderate D3
    
  Archetype 6: "At-Risk Disengaged"
    Low D5 (Behavioral) + High D7 (Dropout Risk) + Low D8 (Social)

CLUSTER USAGE RULES:
  Cluster assignment is INFORMATIONAL only
  Cluster labels cannot gate academic programs or opportunities
  Cluster data used ONLY for: intervention design, curriculum planning,
    engagement strategy, recruiter talent pool intelligence
  Cluster assignment reviewed and re-evaluated monthly
  Students can view own cluster assignment — never other students' clusters
```

## AXIS D10 — Longitudinal Cohort Shift Axis (LCSA)

**What it produces:** Batch-over-batch distribution shift analysis — how the student population is changing over academic years.

**Source:** Historical `PopulationDistributionSnapshot` records

**Computation:**

```
LCSA_delta(axis, cohort_A, cohort_B) = {
  median_shift:    CohortB_median(axis) − CohortA_median(axis)
  std_shift:       CohortB_std(axis) − CohortA_std(axis)
  at_risk_delta:   CohortB_at_risk_rate − CohortA_at_risk_rate
  gifted_delta:    CohortB_gifted_rate − CohortA_gifted_rate
  dropout_delta:   CohortB_dropout_risk_rate − CohortA_dropout_risk_rate
  significance:    Mann-Whitney U test p-value
}

Significance Gate:
  IF p-value < 0.05 → shift is statistically significant → highlight
  IF p-value ≥ 0.05 → shift may be noise → show with uncertainty note

Cohort Trajectory Report:
  3-year rolling comparison
  Institute trend: improving / stable / declining per axis
  Compared to: national trend on same axes

Academic Intake Quality Tracking:
  Year-on-year intelligence distribution shift for incoming students
  Identifies: is the institute attracting stronger or weaker intake cohorts?
  Used by: Academic Board for enrollment strategy
  Used by: Institute improvement planning
```

```

```

     Different from at-risk (not declining) — needs challenge, not support
     Recommended: advanced tracks, championship challenges, mentorship

# SECTION H — ML MODEL SPECIFICATION

## H1 — Model Architecture Stack

```
DISTRIBUTION ENGINE:     Kernel Density Estimation (KDE — Gaussian kernel)
                         Parametric fits: Normal, Beta, Log-Normal per axis
                         Mixture models for bimodal distributions

CLUSTERING:              Gaussian Mixture Model (GMM) — MDTCA
                         K-Means for initialization seeding
                         Optimal K via Bayesian Information Criterion (BIC)
                         Soft cluster memberships (probabilistic assignment)

DROPOUT SURVIVAL:        Cox Proportional Hazards Model (primary)
                         Random Survival Forest (validation model)
                         Time-varying covariates supported (activity changes over time)

LONGITUDINAL SHIFT:      Mann-Whitney U test for significance
                         Cohen's d for effect size
                         Bootstrap confidence intervals for median shifts

PERCENTILE NORMS:        Empirical percentile tables from platform population
                         Refreshed quarterly as population grows
                         Smoothed using isotonic regression for monotonicity

EXPLAINABILITY:          SHAP values at:
                           → Segment boundary explanations
                           → Dropout risk factor attribution
                           → Cluster defining feature attribution
```

## H2 — Model Training Governance

```
Clustering (GMM):
  Minimum training population: 500 students
  Re-cluster: monthly (population shifts)
  Stability validation: run 10 times → measure cluster assignment agreement
  Silhouette score gate: ≥ 0.40 or cluster solution rejected

Dropout Survival Model:
  Training data: students with verified outcome (active or dropped)
  Minimum labeled outcomes: 200 (active) + 50 (dropped)
  Validation: 70/30 time-split (train on earlier cohorts, validate on later)
  Performance gate: C-index (concordance) ≥ 0.75

Percentile Norms:
  National norms: quarterly refresh
  State norms: quarterly refresh
  Minimum 1,000 students per norm group
  Confidence interval mandatory on all percentiles

Bias Audit:
  Quarterly: cluster membership by protected attributes
  Quarterly: at-risk rate by protected attributes
  Quarterly: dropout risk rate by protected attributes
  Violation → model hold → governance review → remediation before redeploy
```
