# 🔒 SECTION 2 — WHAT THIS MODULE DOES (SYSTEM DEFINITION)

**ANTIGRAVITY Institute Performance Predictor** is the ML-powered composite performance intelligence engine — the third and most comprehensive module in the ANTIGRAVITY system. It consumes signals from both prior ANTIGRAVITY modules and all ECOSKILLER platform engines to build a real-time, longitudinal, multi-dimensional performance model for every institute on the platform.

Its singular operational mandate is:

> **Predict, score, explain, benchmark, and govern the complete performance trajectory of every institute on the ECOSKILLER platform — across student outcomes, faculty quality, placement success, curriculum health, Dojo engagement, revenue sustainability, HR market alignment, and national competitive standing — and surface early warning signals before performance degradation becomes irreversible.**

This module answers five core performance intelligence questions at all times:

```
QUESTION 1 — WHERE IS THIS INSTITUTE PERFORMING RIGHT NOW?
  A real-time composite performance score across 8 performance dimensions,
  updated continuously from live platform signals.

QUESTION 2 — WHERE WILL THIS INSTITUTE BE IN 30 / 90 / 180 DAYS?
  A predictive performance trajectory using multi-dimensional ML forecasting,
  including optimistic, baseline, and pessimistic scenarios.

QUESTION 3 — WHAT IS CAUSING PERFORMANCE DEGRADATION?
  A root-cause attribution engine that traces every performance score drop
  to its originating signal — whether curriculum, faculty, student behavior,
  market shift, or operational failure.

QUESTION 4 — HOW DOES THIS INSTITUTE COMPARE TO PEERS?
  A fair, anonymized, domain-normalized national benchmarking system that
  places every institute in a competitive performance band.

QUESTION 5 — WHAT MUST THIS INSTITUTE FIX AND IN WHAT ORDER?
  A prioritized, evidence-based intervention recommendation system that
  ranks every corrective action by predicted performance recovery impact.
```

# 🔒 SECTION 5 — ML MODEL ARCHITECTURE (LOCKED)

## 5A. Core Performance Prediction Models

```
MODEL_P1: COMPOSITE_PERFORMANCE_SCORER
  Type          : Weighted Ensemble Model
                  (Gradient Boosted Trees per dimension +
                   Learned weight optimizer via Bayesian Optimization)
  Input         : All 8 dimension snapshots (real-time) +
                  curriculum_health_feed (Gap Detector) +
                  revenue_health_feed (Revenue Model)
  Output        : composite_performance_score (0–100)
                  per_dimension_score[8] (0–100 each)
                  dimension_weight_applied[8]
                  score_confidence (0.0–1.0)
                  performance_tier (ELITE | HIGH | STANDARD | DEVELOPING | AT_RISK | CRITICAL)
  Trigger       : Run daily per institute
                  Triggered on: any dimension score change > 5 points
  Retrain       : Monthly (dimension weights re-optimized quarterly)

MODEL_P2: PERFORMANCE_TRAJECTORY_FORECASTER
  Type          : Multi-horizon Multi-scenario Time-series
                  (Temporal Fusion Transformer — TFT)
  Input         : Composite performance score history (12 months minimum) +
                  all 8 dimension histories + seasonal signals +
                  governance_event_log + market_shift_signals
  Output        : trajectory_optimistic_30d / 90d / 180d
                  trajectory_baseline_30d / 90d / 180d
                  trajectory_pessimistic_30d / 90d / 180d
                  inflection_point_prediction (date + driver)
                  recovery_probability (0.0–1.0) if currently AT_RISK
                  degradation_velocity (points/week)
  Trigger       : Run weekly per institute
  Retrain       : Quarterly (minimum 18-month history required)

MODEL_P3: ROOT_CAUSE_ATTRIBUTION_ENGINE
  Type          : Causal Graph Model (DAG-based Attribution —
                  SHAP values on XGBoost + Causal Discovery via PC Algorithm)
  Input         : All 8 dimension scores + score deltas (30d window) +
                  event log (governance, curriculum, match, placement events) +
                  ANTIGRAVITY gap signal feed + revenue signal feed
  Output        : primary_cause_of_change (ranked top 5)
                  cause_attribution_weight (% contribution per cause)
                  causal_chain_path (event → dimension → composite)
                  confidence_of_attribution (0.0–1.0)
                  evidence_node_list[]
  Trigger       : Run on every composite score change > 3 points
                  Run on PERFORMANCE_TIER_CHANGE event
  Retrain       : Quarterly (causal graph re-learned from new event data)

MODEL_P4: STUDENT_OUTCOME_PREDICTOR
  Type          : Multi-task Learning Model
                  (Shared-bottom neural network — 3 task heads)
  Task Head 1   : Dropout probability (classification)
  Task Head 2   : Belt achievement probability next 90d (classification)
  Task Head 3   : Placement probability within 180d (regression)
  Input         : STUDENT_PERFORMANCE_SNAPSHOT + STUDENT_HABIT_SIGNAL +
                  COHORT_OUTCOME_AGGREGATE + mentor calibration score +
                  curriculum health score (Gap Detector)
  Output per student:
                  dropout_probability (0.0–1.0)
                  belt_achievement_probability_90d (0.0–1.0)
                  placement_probability_180d (0.0–1.0)
                  outcome_risk_level (HIGH | MEDIUM | LOW)
                  recommended_intervention[]
  Cohort rollup: pct_high_dropout_risk
                  pct_high_placement_probability
                  avg_outcome_risk_level
  Trigger       : Run weekly per student → rolled up to institute
  Retrain       : Monthly

MODEL_P5: FACULTY_MENTOR_PERFORMANCE_PREDICTOR
  Type          : Survival Analysis + Regression (Cox PH + Random Forest)
  Input         : MENTOR_PERFORMANCE_SNAPSHOT + TEACHING_EFFECTIVENESS_SIGNAL +
                  FACULTY_GOVERNANCE_COMPLIANCE + calibration_feed (Gap Detector)
  Output        : mentor_performance_score (0–100)
                  mentor_tier (MASTER | PROFICIENT | DEVELOPING | AT_RISK | SUSPENDED)
                  student_improvement_rate_forecast_90d
                  re_certification_risk_flag (BOOLEAN)
                  bias_escalation_probability (0.0–1.0)
                  recommended_development_action[]
  Institute rollup:
                  avg_faculty_performance_score
                  pct_mentors_at_risk_or_suspended
                  faculty_performance_trend_90d
  Trigger       : Run weekly per mentor → rolled up to institute
  Retrain       : Monthly

MODEL_P6: EARLY_WARNING_SYSTEM (EWS)
  Type          : Multi-signal Anomaly + Threshold Classification
                  (Ensemble: Isolation Forest + Rule-based triggers +
                   CUSUM sequential change detection)
  Input         : All 8 dimension scores (rolling 30d delta) +
                  all ANTIGRAVITY module feeds +
                  PENALTY_FLAG_LOG + governance_compliance_snapshot
  Output        : ews_alert_active (BOOLEAN)
                  ews_severity (CRITICAL | HIGH | MEDIUM | WATCH)
                  ews_signal_type (see SECTION 6 — EWS Signal Taxonomy)
                  ews_dimensions_affected[]
                  ews_days_to_critical_if_unchanged (estimated)
                  ews_recommended_actions[] (priority-ranked)
  Trigger       : Run daily · Event-triggered on any CRITICAL signal
  Retrain       : Monthly

MODEL_P7: NATIONAL_BENCHMARKING_ENGINE
  Type          : Percentile Ranking + Cluster Analysis
                  (K-Means domain clustering + Percentile normalization

  Input         : composite_performance_score (all institutes) +
                  per_dimension_scores (all institutes) +
                  institute_domain_track + institute_tier_type +

  Output        : national_percentile_rank (0–100th)

                  peer_cluster_percentile_rank
                  strongest_dimension_vs_peers
                  weakest_dimension_vs_peers

  Trigger       : Run weekly (aggregate, anonymized)

MODEL_P8: INTERVENTION_PRIORITY_RANKER
  Type          : Multi-objective Optimization
                  (Pareto-optimal action ranking — NSGA-II algorithm)
  Objectives    : Maximize predicted performance recovery ·
                  Minimize time to measurable improvement ·
                  Minimize intervention cost/effort
  Input         : ROOT_CAUSE_ATTRIBUTION (MODEL_P3 output) +
                  EWS signal list (MODEL_P6 output) +
                  all dimension performance gap sizes +
                  historical intervention outcome log
  Output        : ranked_interventions[] (priority-ordered)
                  per intervention:
                    action_type
                    target_dimension
                    expected_recovery_points (performance score)
                    expected_recovery_days
                    effort_level (HIGH | MEDIUM | LOW)
                    evidence_basis[]
                    owner_role (INSTITUTE_ADMIN | CURRICULUM_TEAM |
                               MENTOR | GOVERNANCE_BOARD | PLATFORM_ADMIN)
  Trigger       : Run on every EWS alert + weekly refresh
  Retrain       : Quarterly (outcome feedback loop)

MODEL_P9: PEER_COMPARISON_PERFORMANCE_ANALYZER
  Type          : Differential Analysis Model

  Input         : national_benchmarking_engine cluster assignment +
                  per_dimension_scores + peer cluster dimension scores
  Output        : performance_vs_peer_delta[8 dimensions]
                  dimensions_above_peer_average[]
                  dimensions_below_peer_average[]
                  peer_gap_closure_time_estimate (if below)
                  peer_advantage_sustainability_score (if above)
  Note          : Peer institute identities are NEVER exposed.
                  All comparisons use anonymized cluster statistics only.
  Trigger       : Run weekly (tied to benchmarking run)
  Retrain       : Monthly

MODEL_P10: LONGITUDINAL_PERFORMANCE_TRACKER
  Type          : Change Point Detection + Trend Decomposition

  Input         : Full composite performance history (all available periods) +

  Trigger       : Run monthly
```
