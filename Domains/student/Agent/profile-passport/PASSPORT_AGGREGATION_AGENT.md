## 2️⃣ PURPOSE DECLARATION

### What Problem This Agent Solves

Across the Ecoskiller Antigravity platform, a user's verified identity, earned credentials, skill achievements, belt progressions, project completions, Dojo performance records, and trust scores are generated by multiple independent microservices. These live in isolated data silos — the Skill Engine, Belt Engine, Project Execution Engine, Dojo Match Engine, Job Portal, ERP Layer, and Gamification Engine.

The **PASSPORT_AGGREGATION_AGENT** solves the critical problem of fragmented identity by serving as the single authoritative compiler that:

1. Continuously listens to structured events from all contributing source agents
2. Aggregates verified signals into a unified, tamper-evident **User Passport Object**
3. Computes a composite **Trust Score**, **Skill Depth Index**, and **Domain Completeness Score**
4. Publishes the passport to downstream consumers (Recruiters, Institutes, Job Match Engine, Public Profile, Compliance Layer)
5. Ensures the passport is always consistent, versioned, auditable, and domain-isolated

The User Passport is the single most important credential object on the platform. It represents a user's verifiable professional identity. This agent is the keeper of that truth.

### What Input It Consumes
Structured events from all contributing upstream agents — see Section 3.

### What Output It Produces
A versioned, signed, confidence-scored **User Passport Object** — see Section 4.

### Downstream Agents That Depend on It
- `JOB_MATCH_AGENT` — uses Passport for eligibility scoring
- `RECRUITER_DASHBOARD_AGENT` — displays Passport to employers
- `RANK_ENGINE_AGENT` — uses Passport signals for leaderboard ranking
- `COMPLIANCE_AUDIT_AGENT` — validates Passport integrity
- `FEATURE_STORE_AGENT` — receives Passport-derived feature vectors
- `IDEA_DNA_AGENT` — receives innovation passport segment
- `ROYALTY_ENGINE` — references Passport for royalty attribution
- `PUBLIC_PROFILE_AGENT` — renders SEO-indexed public profile (React/Next.js)
- `PARENT_VISIBILITY_AGENT` — reads passport in read-only trust layer
- `NOTIFICATION_AGENT` — triggers alerts on passport milestone events
- `OBSERVABILITY_AGENT` — monitors passport health metrics

### Upstream Agents That Feed It
- `BELT_ENGINE_AGENT` — belt level, promotion events
- `MATCH_SCORING_AGENT` — Dojo match results, win/loss records
- `SKILL_GAP_AGENT` — skill completion signals
- `PROJECT_MILESTONE_AGENT` — milestone completion, evidence verification
- `CERTIFICATION_AGENT` — issued certification records
- `GAMIFICATION_AGENT` — XP, streaks, badges, achievements
- `JOB_APPLICATION_AGENT` — application history, offer records

```yaml
MODEL_TYPE: Multi-signal Classification + Regression Ensemble
```

## 7️⃣ SECURITY ENFORCEMENT (ZERO-TRUST — ALL MANDATORY)

```yaml
TENANT_ISOLATION:
  - Every passport query and write is scoped by tenant_id
  - Row-level security enforced at PostgreSQL level
  - No wildcard tenant queries permitted
  - tenant_id is NEVER derived from user-supplied input — always resolved from JWT claim

DOMAIN_ISOLATION:
  - Passport data is scoped to user's registered domain_track
  - Cross-domain skill or match signals REJECTED and logged
  - Domain mixing = SECURITY_FAILURE event emitted to COMPLIANCE_AUDIT_AGENT

ROLE-BASED AUTHORIZATION:
  - STUDENT: Read own passport only
  - RECRUITER: Read candidate passport (with explicit candidate consent flag)
  - INSTITUTE ADMIN: Read passports of enrolled students within tenant
  - PLATFORM ADMIN: Read-only access with full audit trail
  - PARENT: Read-only trust layer view (stripped PII fields)
  - AUTOMATION AGENT: Write access limited to registered source_agent identities only
  - No role may write directly to UPO — only this agent may write

ENCRYPTION:
  - PII fields encrypted at rest (AES-256)
  - Passport signature (HMAC-SHA256) verified on every read
  - All inter-service communication over mTLS
  - Secrets via Secret Manager only (no plaintext env vars in production)

AUDIT_LOGGING:
  - Every input event logged to immutable audit log (append-only PostgreSQL partition)
  - Every passport emission logged with full decision path
  - Every read of a passport logged with actor_id and timestamp
  - Every anomaly_flag logged with source signal details
  - Logs are NEVER deletable by any agent including this one
```

## 9️⃣ FAILURE POLICY (NON-NEGOTIABLE — ALL CASES DEFINED)

```yaml
FAILURE_CASES:

  INVALID_INPUT:
    Action: REJECT_EVENT
    Log: LOG_VALIDATION_FAILURE (audit_log + dead_letter_queue)
    Escalate: COMPLIANCE_AUDIT_AGENT
    Retry: NO — invalid input is not retried
    Silent_Failure: FORBIDDEN

    Action: STOP_EXECUTION for this user's passport job
    Log: LOG_INCIDENT (OBSERVABILITY_AGENT)
    Escalate: ON_CALL_ENGINEER via PagerDuty
    Retry: YES — exponential backoff, max 3 attempts, 30s/60s/120s

    Action: EMIT passport WITHOUT AI narrative (core UPO unaffected)
    Log: LOG_AI_TIMEOUT
    Escalate: OBSERVABILITY_AGENT (non-critical alert)
    Retry: YES — AI narrative generation retried async, max 2 attempts
    Silent_Failure: FORBIDDEN — timeout must be logged

  DATA_CORRUPTION:
    Action: STOP_EXECUTION
    Log: LOG_DATA_INTEGRITY_FAILURE (critical severity)
    Escalate: COMPLIANCE_AUDIT_AGENT + ON_CALL_ENGINEER
    Retry: NO — corruption must be investigated before retry
    Quarantine: Affected user passport flagged as UNDER_REVIEW

  CONFIDENCE_BELOW_FLOOR (< 0.6):
    Action: HOLD passport — do not emit
    Log: LOG_LOW_CONFIDENCE_HOLD
    Escalate: OBSERVABILITY_AGENT
    User_Notification: Trigger NOTIFICATION_AGENT with "passport incomplete" advisory
    Retry: YES — re-evaluate after next upstream event received

  CROSS_TENANT_SIGNAL_DETECTED:
    Action: REJECT_EVENT + STOP_EXECUTION
    Log: LOG_SECURITY_VIOLATION (critical)
    Escalate: COMPLIANCE_AUDIT_AGENT + PLATFORM_ADMIN
    Retry: NO
    Silent_Failure: FORBIDDEN — this is a security event

  DUPLICATE_EVENT:
    Action: SKIP (idempotency via Redis dedup registry)
    Log: LOG_DUPLICATE_SKIP
    Escalate: None
    Silent_Failure: FORBIDDEN — must log the skip

GLOBAL_RULES:
  - No silent failures under any condition
  - Every failure path must produce an audit log entry
  - ESCALATE_TO: OBSERVABILITY_AGENT (operational) | COMPLIANCE_AUDIT_AGENT (security/integrity)
```
