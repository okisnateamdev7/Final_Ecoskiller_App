### A.1 — Purpose Declaration

The Antigravity Candidate Summary Generator (ACSG) is a sealed, deterministic, role-scoped narrative engine that synthesizes all verified data signals across the ECOSKILLER + Dojo SaaS platform into structured, human-readable candidate summaries. It serves 300 defined user types — from Grade 6 school students to AI agent orchestration managers — each receiving a summary output calibrated precisely to their role, permission level, and use-case context.

ACSG does NOT generate subjective opinions. Every sentence in every summary is anchored to a verified data signal with a traceable evidence reference. Unverified claims cannot appear in any ACSG output. Opinion language is structurally forbidden.

The **Antigravity** principle applies here as a counterforce against:

- Inflated self-descriptions on traditional resumes with no evidence base
- Recruiter snap judgments based on visual CV formatting
- Institutional summaries that obscure poor performance with generic praise
- AI-generated summaries that hallucinate achievements not present in the data
- Summary templates that encode cultural, linguistic, or demographic bias

ACSG produces **evidence-anchored narrative** — structured truth from verified data, rendered appropriately for every audience in the ecosystem.

## 🔒 SECTION B — ANTIGRAVITY SUMMARY MECHANICS

### B.1 — The Evidence Anchor Rule (Immutable)

Every sentence in every ACSG output must satisfy the Evidence Anchor Rule:

```
EVIDENCE ANCHOR RULE:
  Claim → must map to at least one of:
    (a) Verified match result (Scoring Engine)
    (b) Belt/certification record (Belt Engine + Certification Engine)
    (c) AISA Semantic Intelligence Report (AISA v1.0)
    (d) APPM promotion probability record (APPM v1.0)
    (e) EIE-verified real-work signal (GitHub, Jira, Salesforce, etc.)
    (f) Assessment result (HackerRank, Codility, TestGorilla)
    (g) Course completion record (Education Engine)
    (h) Championship result (Tournament Engine)
    (i) Employer feedback signal (T13 Outcome Validation)
    (j) Mentor evaluation (Mentor Control Engine)

  Claims without evidence anchor → BLOCKED
  Claims with low-confidence evidence (< 0.70) → flagged, not blocked
    but must carry confidence disclosure in output
  Opinion language → STRUCTURALLY FORBIDDEN in all tiers except
    Mentor Qualitative Panel (governed separately — Section F.4)
```

### B.2 — What "Antigravity" Means in Summary Generation

Traditional candidate summaries fail because they:

1. Reproduce whatever the candidate claims (resume = self-marketing, not evidence)
2. Compress genuine achievement behind generic phrases ("team player," "results-driven")
3. Hide gaps and weaknesses that employers need to see
4. Encode evaluator bias in subjective language
5. Provide zero traceability back to actual performance data

The ACSG **applies antigravity** to each of these failure modes:

| Failure Mode | Antigravity Countermeasure |
|-------------|--------------------------|
| Unverified self-claims | Evidence Anchor Rule blocks all unverified content |
| Generic praise language | Structured dimension scores replace adjectives |
| Gap hiding | Explicit gap disclosure module (Section D.5) |
| Evaluator bias language | Bias-controlled narrative templates (Section E) |
| Zero traceability | Every claim carries a data source reference ID |

### D.3 — Gap Disclosure Module (Mandatory — Cannot Be Suppressed in Candidate View)

Per the Antigravity principle, gap disclosure is a non-negotiable feature. A summary that only shows strengths is marketing, not intelligence.

```
Gap Disclosure Rules:
  - Top 3 skill gaps MUST appear in Candidate Self-View (View 1)
  - Gap data is NEVER shown in Employer Hiring View (View 2)
  - Gap data appears in Institution View (View 3) with developmental framing
  - Gap data appears in Mentor View (View 4) for evaluation preparation
  - Gap data in Government/NGO View (View 5) as aggregate only

Gap Computation:
  Gap = benchmark_score_for_level - candidate_score_for_dimension
  Benchmark source: Scoring Engine cohort percentile data
  Gap threshold for disclosure: gap > 10 percentile points vs level benchmark

Gap Narrative Rules:
  - Framed as "Areas to develop" — never as "failures" or "weaknesses"
  - Each gap includes recommended next action (scenario type, drill, course)
  - Gap trend included: widening / stable / closing
```

---

### D.4 — Confidence Disclosure Rules

When evidence confidence is below threshold, the summary must disclose:

| Confidence Level | Disclosure Treatment |
|-----------------|---------------------|
| High (≥ 0.85) | No disclosure needed — clean summary |
| Medium (0.70–0.84) | Footnote: "Based on [N] sessions; additional verification recommended" |
| Low (0.55–0.69) | Inline flag: "⚠ Limited evidence — summary may not fully represent current level" |
| Very Low (< 0.55) | Summary section suppressed; replaced with: "Insufficient verified data for this section" |

## 🔒 SECTION E — BIAS-CONTROLLED NARRATIVE ENGINE

### E.1 — Narrative Template Architecture

All ACSG narrative output is generated from structured templates — not free-form AI generation. Templates are versioned, bias-audited before activation, and governed per T8 Content Governance Lock.

**Template Categories (Locked):**

```
TMP-01: Strength Statement Template
  Pattern: "{candidate} demonstrated [skill_dimension] at [band_label] level,
            evidenced by [evidence_count] verified [source_type] sessions."
  Forbidden substitutions: adjectives not derived from score data

TMP-02: Trend Statement Template
  Pattern: "{skill_dimension} performance has [improved/stabilized/declined]
            by [delta_band] over the past [time_window], based on [session_count] sessions."
  Forbidden: predictions presented as facts

TMP-03: Achievement Statement Template
  Pattern: "Completed [certification_name] (v[version]) on [date],
            assessed across [scenario_count] scenarios with [confidence_label] confidence."
  Forbidden: superlatives ("excellent", "outstanding") unless mapped to top-quartile data

TMP-04: Gap Developmental Statement Template (Candidate View only)
  Pattern: "Developing [skill_dimension]: current performance at [band_label] level;
            [N] targeted sessions in [scenario_type] recommended for progression."
```

### E.2 — Forbidden Language Registry (Locked)

The following language patterns are blocked by the Bias Filter Engine before any narrative renders:

**Category 1 — Demographic Bias Language (Hard Block):**
```
Blocked terms and patterns:
  - Any reference to age ("young professional", "mature candidate", "fresh graduate" as negative)
  - Any reference to gender in evaluative context
  - Any reference to nationality, accent, origin in evaluative context
  - Any reference to physical appearance
  - Any reference to disability status in evaluative context (accommodation noted separately)
  - Family status references ("mother of", "married")
```

**Category 2 — Unanchored Superlatives (Hard Block):**
```
Blocked if not mapped to top-quartile data:
  "Exceptional", "Outstanding", "Brilliant", "Remarkable", "Extraordinary"
  "Best", "Top-tier", "World-class", "Elite"
  
Allowed only if: data_percentile ≥ 90 AND evidence_count ≥ minimum threshold
```

**Category 3 — Deficit Framing (Hard Block in All Employer/Public Views):**
```
Blocked in employer, public, government views:
  "Failed to", "Unable to", "Struggled with", "Weak in", "Poor performance",
  "Lack of", "Deficient", "Below standard", "Underperformed"
  
Allowed in: Candidate self-view only, with developmental framing wrapper
```

**Category 4 — Opinion Language (Hard Block in All Views):**
```
Blocked everywhere:
  "I believe", "In my opinion", "It seems", "Appears to be",
  "Probably", "Likely a good fit", "Seems motivated",
  "Has potential" (unless mapped to learning velocity data)
```

**Category 5 — Cultural Exclusivity (Hard Block — T12):**
```
Blocked:
  Any scenario-specific reference that implies cultural familiarity as skill
  Any language that penalizes non-native language patterns
  Any reference to communication style that maps to cultural origin
```
