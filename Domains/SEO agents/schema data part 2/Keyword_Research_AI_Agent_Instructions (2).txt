SECTION 2: CONTENT TYPE CLASSIFICATION (CRITICAL FOR SCHEMA SELECTION)
================================================================================

2.1 ARTICLE TYPE DETERMINATION:

   A. NewsArticle Schema Triggers:
      - Look for keywords indicating recent events (dates within last 7-14 days)
      - Time-sensitive keywords: "breaking", "latest", "today", "yesterday", "just announced"
      - Event-specific keywords with specific dates
      - Keywords related to current affairs, incidents, announcements
      - Example: "volcano eruption November 25", "latest tech announcement"
      
      RULE: If content has expiration date or time-sensitive nature = NewsArticle

   B. Article Schema (Evergreen) Triggers:
      - Keywords for timeless information
      - Educational content keywords: "how to", "guide", "tutorial", "benefits", "tips"
      - Keywords that remain relevant indefinitely
      - Health, lifestyle, general knowledge topics
      - Example: "jaggery chana benefits", "best practices", "complete guide"
      
      RULE: If content has no expiration and stays relevant = Article

   C. BlogPosting Schema Triggers:
      - Personal blog-style keywords
      - Opinion-based keywords: "review", "experience", "thoughts on"
      - First-person content indicators
      - Personal website/blog publications
      
      RULE: If content is published on personal blog platform = BlogPosting

2.2 CONTENT FRESHNESS ASSESSMENT:
   - Identify date-specific keywords in the research file
   - Flag keywords with temporal indicators
   - Classify as "time-sensitive" or "evergreen"
   - Recommend schema type based on freshness analysis

SECTION 4: SEMANTIC CONNECTION & ENTITY RELATIONSHIP KEYWORDS
================================================================================

4.1 ENTITY LINKING STRATEGY:

   The most critical aspect of schema implementation is creating SEMANTIC CONNECTIONS
   between entities. Your keyword selection must support these connections.

   A. IDENTIFY RELATIONSHIP KEYWORDS:
      - "works for" relationships: employee-employer keywords
      - "founder" relationships: founder-company keywords  
      - "author" relationships: writer-publication keywords
      - "member of" relationships: person-organization keywords
      - "parent organization" relationships: subsidiary-parent company
      - "same as" relationships: alternative names, social profiles

   B. CROSS-ENTITY KEYWORD MAPPING:
      Example: Hospital Website
      
      Primary Entity: LocalBusiness (Hospital)
      Keywords: "hospital name", "medical center", "healthcare facility"
      
      Secondary Entity: Organization (Hospital as organization)
      Keywords: "established", "founded by", "parent company"
      
      Tertiary Entity: Person (Founder/Doctor)
      Keywords: "doctor name", "founder", "chief physician", "specialist"
      
      Connection Keywords: "founded by Dr. X", "works at Y hospital", "affiliated with"

   C. SOCIAL MEDIA PROFILE KEYWORDS:
      - Identify keywords that indicate social presence
      - Map to "sameAs" property in schema
      - Platform-specific keywords: "LinkedIn profile", "Twitter handle", "Facebook page"
      - Required for both Organization and Person schemas

4.2 AUTHORITY & CREDIBILITY KEYWORDS:
   - Expertise keywords: "expert", "certified", "accredited", "award-winning"
   - Trust indicators: "verified", "official", "established", "trusted"
   - Credential keywords: "licensed", "registered", "qualified"
   - These support E-E-A-T signals in schema markup

SECTION 5: KEYWORD RESEARCH FILE ANALYSIS METHODOLOGY
================================================================================

5.1 STEP-BY-STEP ANALYSIS PROCESS:

   STEP 1: Initial File Review
   - Load the keyword research file provided by SEO Orchestrator
   - Identify total number of keywords in the file
   - Categorize by search volume, competition, relevance
   - Flag any unusual or irrelevant keywords

   STEP 2: Module Alignment Check
   - Read the module specification file
   - Understand the module's primary purpose
   - Identify the main user features
   - Match keywords to feature requirements

   STEP 3: Content Type Classification
   - Analyze keywords for temporal indicators
   - Classify as evergreen vs. time-sensitive
   - Determine primary schema type needed
   - Identify all secondary schema types applicable

   STEP 4: Entity Extraction
   - Extract entity keywords (people, organizations, places)
   - Identify proper nouns and named entities
   - Map entities to potential schema types
   - Create entity relationship map

   STEP 5: Semantic Grouping
   - Group keywords by semantic theme
   - Create keyword clusters for each schema element
   - Identify supporting keywords for each primary keyword
   - Build long-tail variations

   STEP 6: Schema Property Mapping
   - Map keyword groups to specific schema properties
   - Ensure coverage for all required properties
   - Identify keywords for optional but beneficial properties
   - Flag any missing keyword coverage

   STEP 7: Priority Ranking
   - Rank keywords by relevance to module
   - Consider search volume and competition
   - Prioritize keywords supporting multiple schema types
   - Create tiered keyword list (primary, secondary, tertiary)

5.2 KEYWORD FILTERING CRITERIA:

   INCLUDE keywords that:
   - Directly relate to module functionality
   - Support required schema properties
   - Have clear user intent matching module purpose
   - Enable semantic entity connections
   - Have reasonable search volume (if data available)
   - Match content type classification

   EXCLUDE keywords that:
   - Are completely irrelevant to module
   - Contradict the schema type needed
   - Have ambiguous meaning without context
   - Are overly generic without specificity
   - Don't support any schema property
   - Mismatch the content freshness requirement

8.2 COMMON ERRORS TO AVOID:

ERROR 1: Mixing evergreen and time-sensitive keywords incorrectly
- Don't classify breaking news with evergreen keywords
- Don't add date-specific keywords to timeless content

ERROR 2: Insufficient entity keywords
- Always include keywords for people, organizations, places
- Don't skip "sameAs" supporting keywords

ERROR 3: Ignoring secondary schemas
- Don't focus only on primary content schema
- Remember: one page = multiple schemas

ERROR 4: Generic keyword selection
- Avoid overly broad keywords without specificity
- Don't select keywords that don't support schema properties

ERROR 5: Missing semantic connections
- Don't isolate entity keywords
- Always include relationship keywords (works for, founded by, etc.)

ERROR 6: Incomplete geographic coverage
- For local business, include all address components
- Don't forget service area keywords

ERROR 7: Overlooking social media keywords
- Both Organization and Person schemas need "sameAs" keywords
- Include all social platforms mentioned

8.3 SCHEMA DATA SUPERCHARGING PRINCIPLE:

CRITICAL RULE: More information in schema = Better SEO performance

When selecting keywords, ALWAYS aim for:
- Maximum schema property coverage
- Rich entity descriptions
- Complete relationship mapping
- Comprehensive "sameAs" connections
- Detailed geographic information
- Full founder/person information

Don't limit yourself. If you can add more relevant keywords that support
additional schema properties, DO IT.

Example: 
- Minimum: 5 social media profiles for Organization
- Better: 10-20 social media profiles
- Best: 40-50 verified presence URLs

This "supercharged" approach helps Google better understand and rank the content.

10.2 ENTITY DISAMBIGUATION:

When multiple entities share similar names:

A. Add disambiguating keywords:
   - Include location: "John Smith, New York"
   - Include organization: "John Smith, CEO of XYZ Corp"
   - Include specialty: "Dr. John Smith, Cardiologist"

B. Use unique identifiers:
   - Professional credentials

   - Specific achievements

10.3 MULTI-LOCATION BUSINESS HANDLING:

For businesses with multiple locations:

A. Select keywords for EACH location:
   - Individual address keywords

B. Include aggregate keywords:
   - "multiple locations"
   - "branches across [region]"
   - Service area keywords

C. Map to appropriate schema structure:
   - Main Organization schema
   - Individual LocalBusiness schemas for each location
   - ParentOrganization relationships

10.4 SEASONALITY & TEMPORAL KEYWORDS:

B. Map to appropriate schema dates:
