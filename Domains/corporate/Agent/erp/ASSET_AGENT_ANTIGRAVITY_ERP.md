### 4C. 📉 DEPRECIATION ENGINE

```
PURPOSE = Compute, schedule, and report asset depreciation in compliance with
          Ind AS 16 (PPE) / Ind AS 38 (Intangibles) / Companies Act Schedule II

SUPPORTED_DEPRECIATION_METHODS:

  SLM — Straight Line Method:
    Annual Depreciation = (Cost - Salvage Value) / Useful Life
    Monthly = Annual / 12
    Pro-rata in first and last year (based on capitalisation date)

  WDV — Written Down Value Method:
    Rate derived from: 1 - (Salvage / Cost)^(1/Useful Life)
    Monthly = (Book Value at start of month × Rate) / 12
    WDV never reaches zero; disposed at book value

  UOP — Units of Production Method:
    Rate = (Cost - Salvage) / Total Expected Units
    Monthly Depreciation = Rate × Actual Units Produced in Month
    (For machinery and vehicles — odometer / production count input required)

DEPRECIATION_SCHEDULE_RULES:
  - Schedule generated at registration (full asset life projected)
  - Schedule locked after Finance Admin approval
  - Revision allowed ONLY on impairment event (Ind AS 36) — dual approval required
  - Partial year on acquisition/disposal: pro-rata days calculation
  - Bonus Depreciation (Sec 32 IT Act): agent computes both book + tax depreciation separately

DEPRECIATION_POSTING_FLOW:
  Month-end cron → ASSET_AGENT computes all depreciation entries
         │
  Depreciation run summary generated (per asset, per category, totals)
         │
  Summary pushed to Finance Admin review queue
         │
  Finance Admin: Reviews → MFA → Approves posting
         │
  Depreciation posted to Finance Ledger Service
         │
  Kafka: asset.depreciation.posted
         │
  Asset net book value (NBV) updated in asset master
         │
  Audit log entry (immutable)

AGENT_ROLE = Compute + Schedule + Summarise + Prepare posting
AGENT_CANNOT = Post directly to ledger without Finance Admin approval

DEPRECIATION_REPORTS:
  - Monthly depreciation schedule (per asset)
  - Asset-wise accumulated depreciation
  - Category-wise NBV report
  - Block of assets schedule (IT Act compliance)
  - Impairment test flag (AI advisory — if NBV > recoverable amount)
```

```
    NOT_FOUND    → Asset flagged as MISSING (T+0)
    EXTRA_FOUND  → Unregistered asset found → New registration triggered
    CONDITION_CHANGED → Condition downgrade noted → Maintenance or disposal advisory

MISSING_ASSET_ESCALATION:
  Day 0:  NOT_FOUND flagged → Alert Asset Admin + custodian
  Day 3:  No resolution → Escalate to L5+ manager
  Day 7:  No resolution → Escalate to L6+ + Finance Admin
  Day 14: No resolution → Write-off advisory + police FIR recommendation
  Day 30: No resolution → Write-off workflow initiated (human approval required)

VERIFICATION_REPORT:
  - Matched assets: count + value
  - Missing assets: list + NBV at risk
  - Extra (unregistered) assets: count
  - Condition degraded: list + advisory
  - Overall verification score: % matched / total
  - Discrepancy value: INR value of missing + unregistered assets
  - Report locked once submitted (immutable)
  - Finance Admin notified of NBV at risk

QR CODE SPECS:
  - QR Code: ISO/IEC 18004 compliant
  - Content: Asset ID + Tenant Code + Verification Hash
  - Format: PNG (print) + SVG (digital) stored in object storage
  - Print size: minimum 2.5cm × 2.5cm (legible at 30cm scan distance)
  - Replacement: lost/damaged tag → Asset Admin re-generates + re-affixes
    (re-generation audit logged with reason)
```
